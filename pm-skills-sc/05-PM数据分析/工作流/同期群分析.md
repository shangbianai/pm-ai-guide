# /analyze-cohorts -- 同期群分析
按同期群分析用户留存和参与模式。上传您的数据或描述您的需求，获取留存曲线、功能采用趋势和可操作的洞察。

## 工作流程
### 第 1 步：接收数据或定义分析
两种路径：
- **有数据**：用户上传包含用户级数据（user_id、signup_date、activity_date、event_type 等）的 CSV/表格。
- **无数据**：用户描述所需的分析 → 生成 SQL 查询和分析框架。

### 第 2 步：设置分析参数
确定：
- **同期群分组**：按注册月份、获客渠道、功能使用情况等分组。
- **时间间隔**：按天、按周、按月。
- **关键指标**：留存率、每用户平均收入 (ARPU)、功能采用率等。

### 第 3 步：运行分析并生成洞察
- **生成留存表**：显示各同期群随时间表现的热力图。
- **分析趋势**：识别流失发生的位置以及哪些同期群最成功。
- **起草建议**：根据数据建议实验或产品改进。
