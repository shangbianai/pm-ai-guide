## 产品构思头脑风暴 (现有产品)
针对持续产品探索的多维度构思。从产品经理、设计师和工程师的角度生成想法，并筛选出最佳的五个。
### 上下文
您正在协助一个正在为 **$ARGUMENTS** 进行持续产品探索的产品三人组。
如果用户提供了文件（研究数据、机会树、用户画像），请先阅读。如果他们提到了产品 URL，请使用网页搜索来了解产品。
### 领域背景
**产品三人组 (Product Trio)** (Teresa Torres, *Continuous Discovery Habits*)：产品经理 + 设计师 + 工程师共同协作进行探索。“最好的想法通常来自工程师。”探索不是线性的 —— 如果实验失败，请循环回去。使用**机会解决方案树 (Opportunity Solution Tree)** (Teresa Torres) 将机会 → 解决方案 → 实验进行映射。
### 指导说明
用户将描述其目标、目标细分市场和预期结果。请按照以下步骤操作：
1. **理解机会**：确认产品、目标、市场细分和预期结果。如果有任何模糊之处，请要求澄清。
2. **从三个视角进行构思** —— 每个视角生成 5 个想法：
   - **产品经理**：关注业务价值、战略一致性和客户影响
   - **产品设计师**：关注用户体验、易用性和惊喜感
   - **软件工程师**：关注技术可能性、数据利用和可扩展的解决方案
3. **根据以下标准，从所有视角中筛选出前 5 个想法进行优先级排序**：
   - 与所述战略目标的一致性
   - 对预期结果的潜在影响
   - 可行性和所需的工作量
   - 与现有解决方案的差异化
4. **针对每个排序后的想法**，提供：
   - 清晰的名称和一句话描述
   - 选择理由
   - 需要验证的关键假设
请逐步思考。以清晰、结构化的格式展示想法。
如果输出内容较多，请在用户的工作区中保存为 markdown 文档。
