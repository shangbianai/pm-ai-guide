## 客户访谈提纲
创建一个能够挖掘真实见解而非仅仅是观点的结构化访谈提纲。遵循《妈妈测试》(The Mom Test) 原则 —— 询问他们的生活，而不是你的想法。
### 领域背景
客户访谈是持续探索 (Continuous Discovery) **阶段 1 (探索)** 的来源之一。其他来源包括：利益相关者访谈、使用情况分析、数据分析、调查问卷、市场趋势、SEO/SEM 分析。产品经理需要直接接触用户、利益相关者、工程师和设计师 —— “无需代理人”。**产品三人组 (Product Trio)**（产品经理 + 设计师 + 工程师 —— Teresa Torres）应共同参与探索，而不仅仅由产品经理独立完成。
### 上下文
您正在为关于 **$ARGUMENTS** 的研究准备一份客户访谈提纲。
如果用户提供了文件（用户画像、假设列表、产品简报或之前的访谈笔记），请先阅读。
### 指导说明
1. **明确研究目标**：
   - 团队需要回答哪些具体问题？
   - 这项研究将为哪些决策提供信息？
   - 哪些假设需要验证？
2. **创建包含以下部分的访谈提纲**：
   ### 开场白 (2-3 分钟)
   - 自会介绍并说明目的（学习，而非销售）
   - 设定预期：“答案没有对错之分。我们是来向您的经验学习的。”
   - 征求录音许可（如果适用）
   - 确认可用时间
   ### 热身：上下文与背景 (5 分钟)
   - “请告诉我您的角色以及典型的一天/一周是什么样的。”
   - “您从事 [与产品领域相关的活动] 多久了？”
   - 目标：建立联系并了解他们的背景
   ### 核心探索：待完成的任务 (JTBD) (15-20 分钟)
   **现状与行为**（过去时，具体实例）：
   - “请带我回顾一下您上次 [执行我们正在探索的任务] 的情况。发生了什么？”
   - “您使用了哪些工具或方法？”
   - “花了多长时间？还有谁参与其中？”
   **痛点与挫折**（观察，不要引导）：
   - “其中最困难的部分是什么？”
   - “如果您有一根魔杖，您想改变什么？”
   - “您尝试过如何解决这个问题？结果如何？”
   **预期结果**（使用他们的语言，而非你的）：
   - “在这一领域，对您来说‘好’是什么样的？”
   - “您如何知道这是否运行良好？”
   **付费意愿 / 优先级**（实际投入）：
   - “您目前在这一项上花费了多少时间/金钱？”
   - “您寻找过更好的解决方案吗？发现了什么？”
   - “为了解决这个问题，您愿意放弃什么？”
   ### 追问技巧
   当遇到有趣的话题时使用：
   - **“能再多跟我讲讲吗”** —— 展开任何话题
   - **“为什么？”**（温和地询问 2-3 次）—— 挖掘根本原因
   - **“您能给我举个具体的例子吗？”** —— 从观点转向事实
   - **“接下来发生了什么？”** —— 跟随故事线
   - **“那让您感觉如何？”** —— 捕捉情感强度
   ### 《妈妈测试》规则
   - 询问**他们的生活**，而不是你的想法
   - 询问**过去**，而不是未来（“您会使用 X 吗？”是没用的问题）
   - **少说多听** —— 目标是 80/20 的比例（用户说 80%）
   - 访谈期间**绝不推销**
   - 寻找**强烈的情感** —— 它们预示着真实的痛苦或喜悦
   - **赞美是噪音** —— “那听起来很酷！”并不能告诉你任何信息
   ### 总结 (3-5 分钟)
   - “还有什么是我没问到但您觉得重要的事情吗？”
   - “关于这个问题，我还应该去请教谁？”
   - 感谢他们的时间
   - 分享后续步骤（如有）
3. **定制提纲**：根据具体的产品领域、用户画像和研究目标调整问题。根据访谈时长增加或删除部分。
4. **包含笔记模板**：
   ```
   参与者: [姓名 / ID]
   日期: [日期]
   关键任务: [他们试图完成的目标]
   当前解决方案: [他们目前使用的工具/方法]
   最大痛点: [他们头号的挫折感]
   预期结果: [成功后的样子]
   付费意愿: [他们投入了多少 / 愿意投入多少]
   意外发现: [意料之外的事情]
   跟进: [后续步骤]
   ```
保存为 markdown 格式。包含提纲和笔记模板。
