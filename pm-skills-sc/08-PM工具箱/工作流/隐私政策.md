# /privacy-policy -- 隐私政策生成器

为您的产品起草一份全面的隐私政策。涵盖数据类型、管辖权、合规性（GDPR、PIPL、CCPA），并标记需要法律审查的条款。

## 调用方式

```
/privacy-policy 一个收集用户行为数据的 SaaS 分析工具 —— 服务于中国和欧盟客户
/privacy-policy 一个包含位置数据和第三方集成的移动应用
```

## 工作流程

### 第 1 步：收集背景信息

询问：
- 是什么产品或服务？
- 收集哪些数据？（个人信息、使用数据、Cookie、位置、支付）
- 您的用户在哪里？（决定适用的法规：GDPR、PIPL、CCPA 等）
- 是否有任何第三方数据共享？（分析、广告、集成）
- 数据存储：在哪里存储，存储多久？
- 年龄限制？（COPPA 考虑因素）

### 第 2 步：起草政策

应用 **privacy-policy** 技能：

生成涵盖以下内容的章节：
- 收集哪些数据以及如何收集
- 如何使用数据（目的）
- 处理的法律依据 (GDPR/PIPL)
- 数据共享和第三方
- 数据保留和删除
- 用户权利（访问、删除、迁移、退出）
- Cookie 政策
- 安全措施
- 儿童隐私（如果适用）
- 国际传输
- 联系信息
- 政策更新流程

### 第 3 步：交付

```
## 隐私政策：[产品]

[完整政策文本]

### 合规性检查清单
| 法规 | 状态 | 注释 |
|-----------|--------|-------|

### 需要法律审查的条款
| 条款 | 为什么 | 优先级 |
|--------|-----|----------|

### 实施检查清单
- [ ] Cookie 同意横幅
- [ ] 数据主体请求流程
- [ ] 数据处理记录
- [ ] 与处理者的 DPA
```

保存为 markdown。提供导出为 DOCX 的选项。

## 注意事项

- 这只是一个模板 —— 在发布前应由法律顾问进行审查
- GDPR 和 PIPL 有特定的要求，不能仅靠猜测 —— 标记出必须由专家审查的地方
- 隐私政策应使用通俗语言，而非晦涩的法律术语
- 当数据处理实践发生变化时应更新政策，而不只是每年更新一次
