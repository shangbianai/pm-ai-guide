# /test-scenarios -- 测试场景生成器

将用户故事或功能描述转化为 QA 可立即执行的全面测试场景。涵盖正常路径 (Happy Paths)、边界情况 (Edge Cases)、错误处理 (Error Handling) 以及跨浏览器/设备的兼容性考虑。

## 调用示例

```
/test-scenarios [粘贴用户故事或验收标准]
/test-scenarios [上传 PRD 或功能规格书]
/test-scenarios 用户可以通过电子邮件链接重置密码
```

## 工作流程

### 第 1 步：接收输入

接受：用户故事、验收标准、PRD 章节、功能描述或任何关于预期行为的规格说明。

### 第 2 步：生成测试场景

应用 **test-scenarios** 技能：

针对每个用户故事或需求，生成：
- **正常路径场景 (Happy Path Scenarios)**: 预期的用户流程正常运行
- **边界情况 (Edge Cases)**: 边界条件、异常输入、并发操作
- **错误场景 (Error Scenarios)**: 当出现问题时系统的反应
- **安全场景 (Security Scenarios)**: 如果适用（鉴权、权限、数据访问）
- **性能场景 (Performance Scenarios)**: 如果适用（负载、超时、大数据量）

### 第 3 步：结构化输出

```
## 测试场景: [功能项目]

**来源**: [用户故事 / PRD / 描述]
**总场景数**: [数量]
**覆盖范围**: [正常路径 / 边界情况 / 错误处理 / 安全 / 性能]

### 场景 1: [标题]
**测试内容**: [对应的用户故事或需求]
**前提条件**: [测试前需要的设置]
**用户角色**: [谁在执行此操作]

| 步骤 | 操作 | 预期结果 |
|------|--------|----------------|
| 1 | [用户动作] | [系统的预期响应] |
| 2 | [用户动作] | [系统的预期响应] |

**后置条件**: [完成后的状态]
**优先级**: [极高 (Critical) / 高 (High) / 中 (Medium) / 低 (Low)]

---
[为每个场景重复上述格式]

### 覆盖率矩阵
| 需求项 | 正常路径 | 边界情况 | 错误处理 | 备注 |
|------------|-----------|-----------|---------------|-------|

### 测试数据需求
[执行这些场景需要准备哪些测试数据]
```

保存为 markdown 文件。

### 第 4 步：后续建议

- "需要我为这些场景 **生成测试数据** 吗？"
- "我是否应该为某个特定场景 **添加更多边界情况**？"
- "需要我 **创建这些场景所测试的用户故事** 吗？"

## 注意事项

- 优先考虑正常路径，然后层层叠加边界情况 —— 确保基本流程在测试边界之前能跑通
- 原始故事中的每个验收标准都应映射到至少一个测试场景
- 包含正向测试（功能正常）和负向测试（优雅地处理失败）
- 对于 API，包含频率限制 (Rate Limiting)、超时、格式错误的请求和鉴权失败的场景
- 标记需要特定测试环境或第三方服务模拟 (Mocking) 的场景
