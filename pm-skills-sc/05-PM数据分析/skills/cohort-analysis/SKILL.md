# 同期群分析与留存探索器

## 目的
按同期群分析用户参与和留存模式，以识别用户行为、功能采用和长期参与的趋势。结合定量洞察与定性研究建议。

## 工作原理

### 第 1 步：读取并验证数据
- 接受包含用户同期群信息的 CSV、Excel 或 JSON 数据文件
- 验证数据结构：同期群标识符、时间周期、参与指标
- 检查缺失值和数据质量问题
- 总结关键统计数据（同期群大小、日期范围、可用指标）

### 第 2 步：生成定量分析
- 计算同期群留存率和参与趋势
- 识别留存曲线、流失模式和异常情况
- 计算各同期群的功能采用率
- 计算环比 (MoM) 或周期环比变化
- 如果需要，生成使用 pandas 和 numpy 的 Python 分析脚本

### 第 3 步：创建可视化
- 生成留存热力图（同期群 vs 时间周期）
- 创建显示同期群演进的折线图
- 构建功能采用对比图
- 可视化流失点和参与趋势
- 输出为交互式图表或静态图像

### 第 4 步：识别洞察与模式
- 发现一个或多个显著模式：
  - 特定同期群的早期流失
  - 后期参与度的变化
  - 功能采用集群
  - 季节性或时间趋势
- 强调令人惊讶的发现和偏差
- 比较同期群表现以建立基准

### 第 5 步：建议后续研究
- 推荐定性研究方法：
  - 对流失用户进行针对性的用户访谈
  - 对高参与同期群进行功能使用调查
  - 关键交互模式的会话回放
  - 高留存 vs 低留存同期群的赢单/失单分析 (Win/Loss Analysis)
- 设计后续定量研究
- 建议 A/B 测试或功能实验

## 使用示例

**示例 1：上传 CSV 数据**
```
上传包含以下列的 cohort_engagement.csv：cohort_month, weeks_active,
user_id, feature_x_usage, engagement_score

请求：“分析留存模式，并找出为什么 2025 年 Q4 同期群的表现不如 Q3”
```

**示例 2：描述数据格式**
```
“我有 2025 年 1 月至 12 月的月度用户同期群。每一行显示：
同期群日期、用户 ID、购买频率和支持工单。
分析哪些同期群表现出最好的长期留存。”
```

**示例 3：功能采用分析**
```
上传包含同期群采用数据的 feature_usage.xlsx。

请求：“比较我们新功能在不同同期群中的采用曲线。
哪些同期群采用最快？有什么模式吗？”
```

## 核心能力

- **数据读取**：导入 CSV、Excel、JSON、SQL 查询结果
- **留存分析**：计算并可视化随时间推移的留存率
- **同期群对比**：对比不同同期群组的指标
- **异常检测**：标注异常模式或流失点
- **Python 脚本**：生成可重用的分析代码，用于持续分析
- **可视化**：创建热力图、图表和交互式仪表板
- **研究设计**：建议针对性的后续研究和访谈方法
- **统计摘要**：提供定量指标和相关性分析

## 获得最佳结果的提示

1. **包含时间维度**：提供跨多个时间周期的数据
2. **清晰定义同期群**：明确同期群分组方式（注册月份、功能上线日期等）
3. **提供背景信息**：说明该期间内的产品变更、发布或事件
4. **多个指标**：包含留存、参与度、功能使用、收入等
5. **充足的数据**：至少 3-4 个同期群以识别有意义的模式
6. **请求特定输出**：要求可视化、Python 脚本或研究建议

## 输出格式

您将收到：
- **数据摘要**：同期群概览和数据质量评估
- **定量发现**：关键指标、留存率和趋势分析
- **可视化图表**：留存热力图和趋势图
- **定性建议**：针对性的用户研究和访谈指南
- **后续行动**：基于发现的实验或产品改进建议
