# 需求评分细则

> 供 pm-requirements skill 的 Step 3 使用。每个维度 1-5 分，按关键词和描述特征判断，不依赖 AI 直觉。

## 用户覆盖度（权重 30%）

| 分数 | 判断标准 |
|------|---------|
| 5 | 明确提到"所有X"、"每个X每天"（如"每个门店每天录入"） |
| 4 | 提到"大部分"、"很多"（如"大多数店长需要"） |
| 3 | 提到特定角色，使用频率"每周"或未说明频率 |
| 2 | 特定角色但使用频率很低，或场景不明确 |
| 1 | 没有提到具体用户或频率 |

## 业务影响（权重 30%）

| 分数 | 判断标准 |
|------|---------|
| 5 | 直接关联"收入"、"成本"、"利润"、"销售额" |
| 4 | 关联"效率"、"时间"、"人力"（如"减少统计时间"） |
| 3 | 关联"体验"、"满意度"但无量化指标 |
| 2 | 描述模糊，无法判断实际影响 |
| 1 | 只是"想要"、"希望"，没有说明影响 |

## 紧迫程度（权重 20%）

| 分数 | 判断标准 |
|------|---------|
| 5 | 提到"无法"、"不能"、"已经"（已有明确损失） |
| 4 | 提到"经常"、"反复"、"总是"（反复发生但能凑合） |
| 3 | 没有明确的时间紧迫感 |
| 2 | "未来可能"、"后续需要" |
| 1 | 模糊，看不出是否着急 |

## 技术风险（权重 20%）

> 分数越高 = 风险越低 = 越容易做

| 分数 | 判断标准 |
|------|---------|
| 5 | 无外部系统、API、硬件依赖 |
| 4 | 依赖已有功能（依赖项在功能清单 P0 中） |
| 3 | 依赖外部系统，但已明确"有 API"或"已确认可行" |
| 2 | 依赖外部系统且状态未知（如"需确认 API"） |
| 1 | 涉及未验证技术方案或高度定制化开发 |
