# 01-PM产品探索 (PM Product Discovery)

产品经理产品发现技能库：涵盖构思、实验设计、假设测试、功能优先级排序和客户访谈。

## 技能 (Skills) (13)

- **analyze-feature-requests** — 按主题、战略一致性、影响力、努力程度和风险对功能请求列表进行分析和优先级排序。
- **brainstorm-experiments-existing** — 为现有产品设计实验以测试假设。
- **brainstorm-experiments-new** — 为新产品设计精益创业实验 (Pretotypes)。
- **brainstorm-ideas-existing** — 使用多视角构思（产品经理、设计师、工程师）为现有产品集思广益产品构思。
- **brainstorm-ideas-new** — 为初始发现阶段的新产品集思广益功能构思。
- **identify-assumptions-existing** — 从价值、可用性、可行性和商业生命力四个维度识别现有产品功能构思中的高风险假设。
- **identify-assumptions-new** — 识别新产品构思在 8 个风险类别（包括进入市场、战略和团队）中的高风险假设。
- **interview-script** — 创建结构化的客户访谈脚本，包含 JTBD 探索问题、破冰、核心探索和总结部分。
- **metrics-dashboard** — 定义和设计产品指标仪表盘，包含关键指标、数据源、可视化类型和警报阈值。
- **opportunity-solution-tree** — 构建机会解决方案树 (OST) 以结构化产品发现 —— 将期望的结果映射到机会、解决方案和实验。
- **prioritize-assumptions** — 使用影响力 × 风险矩阵对假设进行优先级排序，并为每个假设建议实验。
- **prioritize-features** — 根据影响力、努力程度、风险和战略一致性对功能构思待办事项进行优先级排序。
- **summarize-interview** — 将客户访谈记录总结为结构化模板，包含 JTBD、满意度信号和行动项。

## 工作流 (Workflows) (5)

- `产品发现.md` — 运行完整的产品发现周期 —— 从构思到假设映射再到实验设计。
- `头脑风暴.md` — 从产品经理、设计师和工程师的角度集思广益产品构思或实验 —— 适用于现有或新产品。
- `用户访谈.md` — 准备客户访谈脚本或将访谈记录总结为结构化洞察。
- `指标设置.md` — 设计产品指标仪表盘，包含北极星指标、输入指标、健康指标和警报阈值。
- `需求优先级排序.md` — 分析、分类并对来自客户或利益相关者的一批功能请求进行优先级排序。
