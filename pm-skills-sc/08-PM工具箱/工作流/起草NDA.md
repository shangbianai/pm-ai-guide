# /draft-nda -- 保密协议 (NDA) 起草

根据您的具体情况起草一份专业的保密协议。涵盖信息类型、管辖权、期限，并清晰标记需要法律审查的条款。

## 调用方式

```
/draft-nda 我们初创公司与潜在企业客户之间的双向 NDA
/draft-nda 针对访问我们代码库的自由职业承包商的单向 NDA
```

## 工作流程

### 第 1 步：收集背景信息

询问：
- 参与方是谁？（公司名称和角色）
- 是双向 NDA 还是单向 NDA？
- 需要保护哪些信息？（商业秘密、代码、业务数据、客户数据）
- 管辖权？（适用法律所在的省/州/国家）
- 期限？（保密义务应持续多久）
- 任何特定的疑虑？（竞业禁止、禁止招揽、知识产权归属）

### 第 2 步：起草 NDA

应用 **draft-nda** 技能：

生成一份完整的 NDA，涵盖：
- 参与方和叙述部分
- 保密信息的定义（带有具体示例）
- 接收方的义务
- 排除项（公开知识、独立开发等）
- 期限与存续
- 材料的退回/销毁
- 救济措施
- 适用法律与管辖权
- 标准格式条款（可分割性、完整协议、修订）

### 第 3 步：交付

```
## 保密协议 (NDA)

[带有标记章节的完整 NDA 文本]

### 需要法律审查的条款
| 条款 | 为什么需要审查 | 考虑因素 |
|--------|-------------------|--------------|

### 通俗语言总结
[用简单的术语为非法律专业人士解释此 NDA 的含义]
```

保存为 markdown。提供导出为 DOCX 以供签署的选项。

## 注意事项

- 这只是一个起点 —— 始终建议由合格的法律顾问进行审查
- 使用 ⚠️ 标记任何涉及重大法律风险的条款
- 包含通俗语言注释，以便非法律专业人士理解他们同意的内容
- 通常首选双向 NDA —— 它们更公平且谈判速度更快
