# /write-query -- SQL 查询编写器
为任何数据仓库（BigQuery、PostgreSQL、Snowflake 等）生成生产就绪的 SQL。只需描述您的需求或您试图回答的问题。

## 工作流程
### 第 1 步：理解数据需求
明确：
- **目标**：我们要回答什么问题？（例如，“使用过功能 X 的用户的 30 天留存率是多少？”）
- **数据源**：应该使用哪些表或架构？
- **方言**：BigQuery、PostgreSQL、MySQL 等。

### 第 2 步：起草查询
- **结构**：使用 CTE（通用表表达式）以提高可读性。
- **优化**：使用性能最佳实践（及早过滤，避免使用 `SELECT *`）。
- **文档**：添加注释以解释复杂逻辑。

### 第 3 步：提供最终 SQL 和说明
- **交付查询**：针对目标方言进行格式化。
- **解释逻辑**：数据是如何转换和计算的。
- **验证**：建议如何测试查询的准确性。
