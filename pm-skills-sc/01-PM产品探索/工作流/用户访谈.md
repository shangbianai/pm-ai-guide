# /interview -- 客户访谈准备与总结 (Customer Interview Prep & Summary)

两种模式：**prep** 在与客户交谈前创建结构化的访谈提纲，**summarize** 在访谈结束后提取见解。

## 调用示例

```
/interview prep 企业级用户的新手引导体验
/interview summarize [粘贴转录文本或上传文件]
/interview                    # 询问你需要哪种模式
```

## 模式说明

---

### 准备模式 (Prep Mode)

根据你的研究问题创建定制的结构化访谈提纲。

#### 工作流程

**第 1 步：了解研究目标**

询问用户：
- 你试图了解什么？（具体的研究问题）
- 你在采访谁？（细分市场、角色、与产品的关系）
- 你有多少时间？（15 分钟、30 分钟、60 分钟）
- 这项研究将为哪些决策提供信息？

**第 2 步：生成访谈提纲**

应用 **interview-script** 技能：

- 遵循《妈妈测试》原则 —— 询问他们的生活，而不是你的想法
- 不提引导性问题，不推销，专注于过去的行为和真实情况
- 按部分组织提纲：

```
## 访谈提纲：[研究主题]

**研究问题**: [我们试图学习什么]
**目标参与者**: [谁]
**时长**: [X] 分钟

### 热身 (3-5 分钟)
[建立联系的问题，了解角色/背景]

### 核心探索 (15-40 分钟)
[JTBD 挖掘、过去的行为、当前工作流、痛点]
- 针对每个问题：问题本身 + 为什么要问 + 追问提示

### 特定主题 (5-10 分钟)
[针对特定功能或概念的定向问题 —— 如果需要]

### 总结 (3-5 分钟)
[开放式结尾、转介绍请求、后续步骤]

### 笔记模板
[预格式化的模板，用于在访谈期间捕捉见解]

### 警示信号 (Red Flags)
[对话偏离轨道或参与者只是出于礼貌而非诚实的迹象]
```

**第 3 步：定制与审查**

- 根据时间调整问题数量
- 针对用户想要测试的特定假设添加追问问题
- 标记可能引导受访者的问题
- 提供可打印版本（保存到工作区的 markdown 文件）

---

### 总结模式 (Summarize Mode)

将访谈转录文本转化为结构化、可操作的见解。

#### 工作流程

**第 1 步：接受转录文本**

接受任何格式：
- **粘贴的文本**：原始转录文本或笔记
- **上传的文件**：文档、文本文件或会议记录导出
- **音频总结**：如果用户描述了谈话内容（而非完整转录）

如果输入的是粗略笔记而非完整转录，请根据现有内容进行处理并说明局限性。

**第 2 步：提取与结构化**

应用 **summarize-interview** 技能：

解析转录文本以识别：
- **参与者概况**：角色、经验水平、细分市场、背景
- **核心 JTBD**：他们试图实现的目标
- **当前解决方案**：他们目前如何解决问题（包括“拼凑”的方案）
- **痛点与挫折**：具体的、带有情感强度的证据
- **预期结果**：他们如何定义成功
- **关键引言**：具有代表性的“原声”见解

**第 3 步：生成总结报告**

```
## 访谈总结：[参与者姓名/ID]

**主要结论**: [一句话总结]

### 客户背景
[角色、公司、他们如何处理 [主题]]

### 待完成的任务 (JTBD)
- [任务 1]: [描述]
- [任务 2]: [描述]

### 痛点与障碍
- [痛点]: [描述其严重程度和频率]

### 关键洞察
- [洞察 1]: [为什么这很重要]

### 推荐行动
- [建议的下一步行动]
```

保存为 markdown 文件。

**第 4 步：综合见解（可选）**

如果用户提供了多次访谈，请询问是否需要汇总分析以识别跨用户的模式。
