# /value-proposition -- 价值主张设计

使用 6 部分 JTBD 模板为产品或功能设计清晰、引人入胜的价值主张。这是 Strategyzer 价值主张画布的一种替代方案，它从客户出发，专注于实际成果。

## 调用方式

```
/value-proposition 为非英语母语人士提供的 AI 写作工具
/value-proposition [上传融资计划书、PRD 或竞争分析]
/value-proposition                    # 询问你的产品
```

## 工作流程

### 第 1 步：了解产品和市场

接受来自以下方面的背景：
- 产品描述（口头或书面）
- 上传的文档（融资计划书、PRD、竞争分析）
- 需要优化的现有价值主张

提出关键问题：
- 产品是做什么的？它是为谁准备的？
- 目前存在哪些替代方案或变通方法？
- 你有哪些客户洞察或研究？

### 第 2 步：构建价值主张

应用 **value-proposition** 技能生成 6 部分模板：

```
## 价值主张：[产品名称]

### 针对 [细分市场]：

1. **谁 (Who)**：[目标用户画像和特征]
2. **为什么 (Why)**：[他们试图完成的工作 (JTBD)，期望的成果]
3. **之前 (What Before)**：[他们目前痛苦的现状 —— 现有工具、摩擦、变通方法]
4. **如何做 (How)**：[你的解决方案 —— 交付价值的具体功能和能力]
5. **之后 (What After)**：[改善后的成果 —— 变得可能的事情]
6. **替代方案 (Alternatives)**：[如果没有你他们会用什么，以及为什么你更好]

### 价值主张陈述
[一句话总结：对于 [人群] 来说，他们 [有某种需求]，[产品名称] 是一个 [类别]，能够提供 [益处]。不同于 [替代方案]，我们 [差异化优势]。]

### 价值主张陈述 (可复用)
- 营销：[...]
- 销售：[...]
- 入职：[...]
```

如果用户有多个细分市场，请为每个细分市场创建一个单独的价值主张。

### 第 3 步：保存并提供后续步骤

保存为 Markdown。提供：
- "是否需要我通过价值曲线 (Value Curve) **将其与竞争对手进行比较**？"
- "我是否应该围绕这个价值主张 **构建完整的战略**？"
- "是否需要我使用它 **创建一个精益画布 (Lean Canvas)** 或 **初创公司画布 (Startup Canvas)**？"
- "我是否应该根据这些价值主张陈述 **生成营销信息**？"

## 备注

- 该模板从客户出发（谁/为什么）并向解决方案推进 —— 与 Strategyzer 的画布不同，后者将产品放在左侧
- 每个价值主张都是针对特定细分市场的 —— 不同的细分市场获得不同的价值主张
- 使用价值曲线（蓝海战略）在关键因素上直观地将你的产品与竞争对手进行比较
- 价值主张是产品战略的一个要素 —— 使用 `/strategy` 获取完整视图
