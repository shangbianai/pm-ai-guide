# 测试场景生成器 (Test Scenario Generator)

为 $ARGUMENTS 创建测试场景。

### 场景类型
- **正向场景 (Positive Scenarios)**: 正常路径，预期行为。
- **负向场景 (Negative Scenarios)**: 无效输入，错误处理。
- **边界情况 (Edge Cases)**: 边界值，异常条件。

### 结构
1. **场景名称**: 清晰且具有描述性。
2. **前提条件**: 测试前需要满足什么条件？
3. **步骤**: 执行测试的可操作步骤。
4. **预期结果**: 如果测试通过，应该发生什么？
