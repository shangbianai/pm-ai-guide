# /analyze-feedback -- 用户反馈分析

将大量的用户反馈（评论、调查、支持工单、NPS 回复）处理成结构化的洞察，包含情感分析和细分层级的模式。

## 调用方式

```
/analyze-feedback [上传 NPS 回复的 CSV 文件]
/analyze-feedback [粘贴应用商店评论或调查回复]
/analyze-feedback [上传支持工单导出文件]
```

## 工作流程

### 第 1 步：接收反馈数据

接收任何格式：
- 包含反馈文本的 CSV/Excel（以及可选的元数据：日期、细分、评分）
- 粘贴的文本（评论、调查回复、Slack 消息）
- 从反馈工具导出的上传文档

询问：
- 这是哪种反馈？（NPS、评论、支持工单、调查等）
- 是否有需要单独分析的细分群体？（用户等级、计划、地理位置）
- 您在寻找什么？（通用主题、特定问题、随时间变化的趋势）

### 第 2 步：分析

应用 **sentiment-analysis** 技能：

- **情感评分**：对每条反馈进行分类（正面、中性、负面）
- **主题提取**：识别反复出现的话题并对相关反馈进行聚类
- **频率分析**：统计每个主题出现的频率
- **细分分析**：按用户细分（如果数据可用）细分情感和主题
- **趋势检测**：如果日期可用，识别情感随时间的变化

### 第 3 步：生成分析报告

```
## 反馈分析报告

**日期**: [今天]
**分析的反馈数量**: [数量] 条
**来源**: [NPS 调查 / 应用评论 / 支持工单 / 等]
**期间**: [日期范围，如果可用]

### 整体情感
- 正面: [X%] | 中性: [Y%] | 负面: [Z%]
- 平均情感得分: [X/10]
- 趋势: [改善中 / 稳定 / 下降中]

### 热门主题
| # | 主题 | 提及次数 | 情感 | 受影响最大的细分群体 |
|---|-------|----------|-----------|----------------------|

### 主题深入分析

#### 主题 1: [名称] — [X] 次提及，[情感]
- **用户在说什么**: [带有代表性引用的总结]
- **根本原因**: [驱动此反馈的因素]
- **影响**: [这如何影响留存、满意度或收入]
- **建议**: [该如何处理]

[针对前 5-8 个热门主题重复此部分]

### 细分分析
| 细分群体 | 数量 | 平均情感 | 热门主题 | 关键差异 |
|---------|--------|-------------|-----------|---------------|

### 值得注意的引用
> "[引用内容]" — [细分群体, 情感]

### 随时间变化的趋势
[如果日期数据可用：显示情感变化的图表数据]

### 可操作的洞察
1. [洞察 + 建议采取的操作]
2. ...

### 差距
[这些反馈没有告诉您什么 —— 建议的后续研究]
```

保存为 markdown。如果输入是结构化数据 (CSV)，还将带有情感评分的增强数据保存为 CSV。

### 第 4 步：提供后续步骤

- "是否需要我根据这些反馈模式 **创建用户画像**？"
- "我是否应该 **将热门主题分类为功能请求**？"
- "是否需要我 **设计访谈脚本** 以深入探讨特定主题？"

## 注意事项

- 情感分析是近似的 —— 标记边缘情况（讽刺、混合情感、非英语文本）
- 主题提取应寻找请求背后的需求，而不仅仅是表面话题
