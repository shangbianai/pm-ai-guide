# /plan-launch -- 进入市场策略 (Go-to-Market Strategy)

从基本原理构建完整的 GTM 计划：识别你的滩头市场，定义理想客户，设计消息传递，选择渠道，并创建发布时间表。

## 调用示例

```
/plan-launch 针对咨询公司的 AI 提案撰写工具
/plan-launch 我们项目管理工具的新企业级版本
/plan-launch [上传 PRD、策略文档或路演 PPT]
```

## 工作流程

### 第 1 步：理解发布背景

询问：
- 你要发布什么？（新产品、新功能、新层级、市场扩张）
- 处于什么阶段？（发布前规划、即将发布、发布后优化）
- 你有现有客户吗？还是从零开始？
- 时间线是怎样的？是否有截止日期？
- 预算限制？团队规模？

### 第 2 步：定义滩头市场细分 (Beachhead Segment)

应用 **beachhead-segment** 技能：

- 针对以下维度评估潜在市场细分：
  - 紧迫痛点（他们对该产品的需求有多迫切）
  - 付费意愿（预算和购买决策权）
  - 可赢取的市场份额（你是否能触达并赢得他们）
  - 推荐潜力（他们是否会告诉其他人）
- 建议单一的最佳起始细分市场并说明理由
- 规划在占领滩头阵地后的相邻扩展市场

### 第 3 步：定义理想客户画像 (ICP)

应用 **ideal-customer-profile** 技能：

- 人口/公司统计：公司规模、行业、地理位置、技术栈
- 行为特征：他们如何发现解决方案、购买流程、决策者是谁
- JTBD：他们“雇佣”你的产品去完成的具体任务
- 当前替代方案：他们今天在用什么，以及为什么不够好
- 准入标准：如何快速识别他们

### 第 4 步：构建 GTM 策略

应用 **gtm-strategy** 技能：

- **定位 (Positioning)**：你如何向该细分市场描述自己
- **消息传递 (Messaging)**：针对不同利益相关者（买家、用户、影响者）的关键信息
- **渠道 (Channels)**：在哪里以及如何触达你的 ICP（按预期 ROI 排序）
- **发布战术**：针对发布前、发布当天和发布后的具体行动
- **定价对齐**：定价如何支持 GTM 方式
- **成功指标**：你将如何知道发布是否成功

### 第 5 步：生成 GTM 计划

```markdown
## 进入市场计划：[产品/功能名称]

**发布日期**：[目标日期]
**类型**：[新产品 / 功能 / 层级 / 市场扩张]

### 滩头市场细分 (Beachhead Segment)
**对象**：[特定的细分市场定义]
**为什么先选他们**：[基于评估标准的理由]
**市场规模**：[TAM/SAM/SOM 估算]

### 理想客户画像 (ICP)
| 属性 | 定义 |
|-----------|-----------|
| 公司规模 | [范围] |
| 行业 | [具体行业] |
| 决策者 | [职位/角色] |
| 关键 JTBD | [需要完成的任务] |
| 当前解决方案 | [他们今天在用什么] |
| 准入信号 | [如何识别他们] |

### 定位与消息传递
**定位陈述**：对于 [对象] 且有 [需求] 的人，[产品] 是一个 [类别]，它能提供 [利益]。不同于 [替代方案]，我们 [差异化优势]。

**按利益相关者的关键消息**：
| 受众 | 消息 | 证明点 |
|----------|---------|------------|

### 渠道策略
| 渠道 | 战术 | 触达 | 成本 | 优先级 |
|---------|--------|-------|------|----------|

### 发布时间表
| 阶段 | 时间 | 行动 | 负责人 |
|-------|--------|---------|-------|
| 发布前 | [日期范围] | [行动列表] | [负责人] |
| 发布周 | [日期范围] | [行动列表] | [负责人] |
```
