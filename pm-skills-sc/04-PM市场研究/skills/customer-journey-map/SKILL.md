## 客户旅程图 (Customer Journey Map)

绘制从认知到拥护的端到端客户体验，识别每个阶段的情感、痛点和改进机会。

### 背景

你正在为 **$ARGUMENTS** 创建客户旅程图。

如果用户提供了文件（访谈转录、调查数据、分析数据、支持工单或现有的旅程图），请先阅读它们。如果提供了 URL，请通过网络搜索了解该产品。

### 说明

1. **定义画像**：谁在经历这段旅程？使用具有特定 JTBD 的画像，而不是通用用户。

2. **绘制旅程阶段**（根据产品进行调整）：

   | 阶段 | 描述 |
   |---|---|
   | **认知 (Awareness)** | 他们最初是如何得知产品的？ |
   | **考虑 (Consideration)** | 他们评估什么？他们比较哪些替代方案？ |
   | **获取 (Acquisition)** | 他们如何注册或购买？ |
   | **新手引导 (Onboarding)** | 产品的初次体验 —— 价值实现时间 (Time to Value) |
   | **参与 (Engagement)** | 定期使用 —— 培养习惯 |
   | **留存 (Retention)** | 是什么让他们不断回来？什么可能导致流失？ |
   | **拥护 (Advocacy)** | 他们何时以及为什么向他人推荐产品？ |

3. **对于每个阶段，记录以下内容**：

   - **触点 (Touchpoints)**：用户与产品、品牌或团队互动的地方（网站、电子邮件、应用内、支持、社交媒体）
   - **用户行为 (User actions)**：他们在此阶段做什么
   - **想法与疑问 (Thoughts & questions)**：他们在想什么（“这值得我花时间吗？”“我该如何……？”）
   - **情感 (Emotions)**：他们的感受（兴奋、困惑、沮丧、欣喜） —— 使用评分量表或表情符号指示
   - **痛点 (Pain points)**：摩擦、困惑、流失风险
   - **机会 (Opportunities)**：如何在此时改进体验

4. **识别关键时刻**：
   - **顿悟时刻 (Aha moment)**：用户首次体验到核心价值的时刻
   - **关键决策点 (Moments of truth)**：他们决定投入还是放弃的决策点
   - **流失触发点 (Churn triggers)**：用户最常流失的地方

5. **创建旅程图表格**：

   | 阶段 | 触点 | 用户行为 | 情感 | 痛点 | 机会 |
   |---|---|---|---|---|---|

6. **推荐优先改进事项**：
   - 哪些痛点对转化或留存的影响最大？
   - 哪些快速获胜 (Quick wins) 可以立即改善体验？
   - 哪些需要更深层的投资但回报最高？

按步骤思考。保存为 markdown 文档。对于视觉旅程图，建议用户以该分析为基础在 Miro 或 FigJam 中创建一个。
