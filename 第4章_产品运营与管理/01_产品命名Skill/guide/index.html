01为什么需要命名方法论

"感觉好听"不是命名理由。一个差的名字会让团队在走廊里用另一个名字代替它。

没有方法论的命名

拍脑袋想几个 → 内部投票 → 选票最多的 → 上线后发现撞名/歧义/没人记得住 → 改又不好改

结果：一个产品有三个名字（文档一个、代码一个、大家嘴里一个）

使用 pm-naming

定义命名对象 → 分析公司风格 → 多风格候选 → 四维度评分 → 风险审查 → 推荐方案

结果：一个名字，经得起所有角色追问（品牌说、法务审、开发用、用户搜）

02四种命名风格

没有绝对正确，只有是否适合场景和体系。Skill 会根据你的公司风格推荐主推方向。

A

功能描述型

"订单管理中心""批量导入""智能巡检"
一看就懂，零解释成本

适合：B 端工具、后台系统、内部产品

B

品牌前缀型

"飞鱼洞察""XM Discover"
建立品牌资产，可体系化扩展

适合：产品线、对外品牌、平台级能力

C

隐喻意象型

"朋友圈""瀑布流""银河"
有记忆点，有故事可讲

适合：C 端产品、品牌功能、营销传播

D

动作引导型

"一键诊断""先买后付""猜你喜欢"
突出用户能获得什么

适合：增长功能、CTA 场景、工具型产品

04四维度评估框架

每个候选名称从四个维度打分（1-5），告别"感觉好听"式的判断。

30%
语义清晰度
不需要解释就能懂吗？
25%
品牌一致性
和已有名字协调吗？
25%
传播力
好记、好说、好搜吗？
20%
风险可控性
有歧义、撞名吗？

传播力三测试

走廊测试

在走廊遇到同事，能自然说出"你用 XX 查一下"吗？

搜索测试

用户会在搜索框输入什么？这个名称能匹配上吗？

缩写测试

这个名称被缩略后会变成什么？缩略后还清晰吗？

05使用流程

06风险审查：四个不能踩的雷

好名字可能死在一个没注意到的风险上。

风险类型检查内容示例等级
语言歧义谐音、方言、缩写、中英文双关"惠普"在方言中谐音"废品"中风险
撞名风险竞品、内部项目、知名产品新产品叫"钉钉助手"就撞了中风险
商标/法律商标局查询、域名、行业管制"百事通"撞 Pepsi 注册商标高风险
文化/地域数字忌讳、地域敏感、文化符号Nova 在西班牙语中 ="不走"高风险
有高风险就一票否决。不要心存侥幸——品牌和法务迟早会发现。
