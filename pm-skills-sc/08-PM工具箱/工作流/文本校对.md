# /proofread -- 语法与流畅度检查

识别文本中的语法、逻辑和流畅度错误。提供具体的、有针对性的修复，而不重写整个文档。

## 调用方式

```
/proofread [粘贴文本]
/proofread [上传文档]
```

## 工作流程

### 第 1 步：接收文本

接收任何形式的文本：粘贴的内容、上传的文档（DOCX、PDF、markdown）或电子邮件草稿。

### 第 2 步：分析

应用 **grammar-check** 技能：

扫描三类问题：

**语法 (Grammar)**：拼写、标点、主谓一致、时态一致、冠词使用
**逻辑 (Logic)**：矛盾、缺乏支持的论点、循环论证、不明确的指代
**流畅度 (Flow)**：生硬的过渡、句子节奏、段落结构、冗余、可读性

### 第 3 步：报告问题

```
## 校对报告

**文本长度**: [词数]
**发现的问题**: [按类别统计的数量]

### 问题详情

#### 1. [类别：语法/逻辑/流畅度]
- **位置**: "[有问题的内容引用]"
- **问题**: [错误原因]
- **修复**: "[修改后的内容]"

[针对每个问题重复此格式]

### 总结
- 语法: [X] 个问题
- 逻辑: [X] 个问题
- 流畅度: [X] 个问题
- 整体质量: [评估]
```

### 第 4 步：提供建议

- “想让我**应用所有修复**并返回清洗后的文本吗？”
- “我应该更详细地**关注某个特定章节**吗？”

## 注意事项

- 修复建议应保持最小化 —— 只更改必要的部分，保留作者的原声
- 对于非母语者，要特别清楚地说明*为什么*建议进行更改
- 不要过度修正风格偏好 —— 错误与风格差异是有区别的
- 对于专业文档，还要检查语气的一致性和对受众的适宜性
