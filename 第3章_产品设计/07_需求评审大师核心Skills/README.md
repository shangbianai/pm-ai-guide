# 第3章 07 · 需求评审大师

> 本节配套 Claude Code Skill：`pm-review`

---

## 本节目标

用 AI 在需求评审前模拟 6 个角色（前端、后端、设计、测试、业务方、老板）对 PRD 或 HTML 原型进行预审，提前发现需求疏漏，并生成专业的应对话术。

核心价值：**不是替代评审会，而是让你在评审会上更专业、更从容。**

## 安装 Skill

```powershell
# 复制 SKILL.md 到 Claude Code Skills 目录
Copy-Item skills/pm-review/SKILL.md "$env:USERPROFILE\.claude\skills\pm-review\SKILL.md"
```

安装后重启 Claude Code，输入 `/pm-review` 即可触发。

## 使用场景

| 场景 | 输入 | 预期产出 |
|------|------|----------|
| PRD 评审前准备 | 粘贴 PRD 全文或要点 | 各角色可能提出的问题 + 应对方案 |
| 原型评审前自查 | 粘贴 `.html` 文件或代码 | 交互状态检查 + 各角色问题 + 改进清单 |
| 混合评审 | PRD + 原型一起 | 交叉验证，暴露文档与原型不一致之处 |
| 话术储备 | 描述评审场景 | 通用应对框架 + 场景专用话术 |

---

## 使用方式

### 基础用法

```
/pm-review
```

然后把 PRD 文本或 HTML 原型粘贴进来，Skill 会自动识别输入类型并开始评审。

### 评审报告示例

打开 `需求评审报告-示例.html` 查看完整效果：
- 仪表盘概览（问题总数、严重度分布、应对覆盖率）
- 按角色筛选（前端/后端/设计/测试/业务方/老板）
- 按严重度筛选（阻塞/重要/建议）
- 每道题展开查看应对方案 + 追问预判 + 兜底话术
- 打印友好，可直接作为评审准备文档

### 输入 HTML 原型时的额外检查

Skill 会自动检查每个页面的 4 种状态覆盖：

| 状态 | 说明 |
|------|------|
| 加载态 | 数据加载中用户看到什么 |
| 空状态 | 无数据时的引导 |
| 错误态 | 网络/接口异常时的处理 |
| 成功态 | 正常数据展示 |

---

## 6 角色关注域速览

| 角色 | 关注什么 | 典型提问 |
|------|---------|----------|
| 前端 | 交互实现、状态管理、性能 | "加载态和错误态有设计吗？" |
| 后端 | 数据模型、API、并发安全 | "并发操作数据一致性怎么保证？" |
| 设计师 | 一致性、交互规范、可用性 | "空状态有没有用户引导？" |
| 测试 | 边界条件、异常路径、兼容性 | "最可能出 bug 的地方在哪？" |
| 业务方 | 业务逻辑、用户价值、运营 | "上线后怎么验证效果？" |
| 老板 | 战略对齐、ROI、时间资源 | "为什么现在做？不做会怎样？" |

---

## 输出格式

生成 `.md` 评审报告后，Skill 会询问是否需要额外格式：

- **A**：仅 .md（默认输出）
- **B**：.md + .docx（Word 文档，适合打印分发）
- **C**：.md + HTML（可视化页面，适合屏幕展示）
- **D**：全部三种
