# /research-users -- 用户研究综合

将原始研究数据转化为可操作的用户画像、行为细分和客户旅程图。接受调查数据、访谈笔记、反馈、分析数据或用于探索性研究的产品描述（无数据时）。

## 调用方式

```
/research-users [上传调查结果、访谈笔记或反馈数据]
/research-users 针对机构的 B2B 项目管理工具 —— 帮助我了解我们的用户
/research-users [粘贴用户反馈或支持工单数据]
```

## 工作流程

### 第 1 步：接收研究输入

接收任何组合的输入：
- 调查回复（CSV、电子表格、粘贴文本）
- 访谈笔记或转录
- 支持工单或功能请求
- 产品分析 / 行为数据
- NPS 或满意度数据
- 产品描述（用于没有数据时的探索性研究）

询问：
- 您拥有哪些研究资料？什么格式？
- 您想了解什么？（谁是我们的用户，他们有何不同，摩擦点在哪里）
- 这些研究将为哪些决策提供信息？（路线图、定位、定价、新手引导）

### 第 2 步：构建画像

应用 **user-personas** 技能：

- 从数据中识别 3-4 个截然不同的画像
- 为每个画像提供：名称、角色、目标 (JTBD)、痛点、收益、行为模式
- 包含意外的洞察 —— 数据中令您感到惊讶的事情
- 注明画像流行度（如果数据允许，每个画像占总体的百分比）

### 第 3 步：细分用户

应用 **user-segmentation** 和 **market-segments** 技能：

- 创建行为细分（而不只是人口统计）
- 为每个细分：规模、JTBD、产品契合度、付费意愿、参与水平
- 识别价值最高和增长最快的细分
- 映射细分到画像（他们如何重叠）

### 第 4 步：绘制客户旅程

应用 **customer-journey-map** 技能：

- 绘制端到端旅程：认知 → 考虑 → 新手引导 → 活跃使用 → 扩张 → 拥护
- 为每个阶段提供：触点、情感、痛点、顿悟时刻
- 识别流失最严重的节点
- 突出值得放大的惊喜时刻

### 第 5 步：生成研究报告

```
## 用户研究报告: [产品]

**日期**: [今天]
**数据源**: [分析了什么]
**样本量**: [如果适用]

### 执行摘要
[3-5 句话：关键发现和影响]

### 用户画像

#### 画像 1: [名称] — "[捕捉其特征的引用]"
- **是谁**: [角色, 背景, 经验水平]
- **核心 JTBD**: [当..., 我想要..., 以便我能够...]
- **关键痛点**: [前 3 个]
- **关键收益**: [令他们欣喜的地方]
- **行为模式**: [他们如何使用产品]
- **流行度**: [占用户群的 X%]

[针对每个画像重复此部分]

### 用户细分
| 细分市场 | 规模 | 核心 JTBD | 产品契合度 | 价值 | 增长 |
|---------|------|-------------|-------------|-------|--------|

### 客户旅程图
| 阶段 | 触点 | 情感 | 痛点 | 机会 |
|-------|------------|---------|-------------|---------------|

### 关键洞察
1. [带有支持证据的洞察]
2. ...

### 建议
1. [与发现相关的可操作建议]
2. ...
```
