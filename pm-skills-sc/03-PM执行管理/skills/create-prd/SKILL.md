# PRD 生成器

为 $ARGUMENTS 创建结构化的产品需求文档 (PRD)。

### PRD 结构
1. **背景与问题**：我们为什么要构建这个功能？我们要解决什么用户痛点？
2. **目标**：成功是什么样子的？
3. **用户故事**：目标用户是谁，他们想要什么？
4. **功能需求**：详细的技术和 UI 规范。
5. **成功指标**：我们将如何衡量影响力？
6. **超出范围**：我们 **不** 构建什么？

### 原则
- **清晰性 (Clarity)**：使用简单的语言。
- **完整性 (Completeness)**：覆盖边界情况和限制条件。
- **对齐 (Alignment)**：确保所有利益相关者都理解 "为什么" (Why)。
