# /business-model -- 商业模式探索

使用四种互补的框架构建和分析商业模式。选择其中一个或运行全部以获得完整视图。

## 调用方式

```
/business-model lean 连接自由职业 PM 与初创公司的市场平台
/business-model full 企业级分析平台
/business-model startup 为非英语母语人士提供的 AI 写作工具
/business-model value-prop SaaS 入职工具
/business-model all SaaS 入职工具        # 运行所有四个框架
/business-model                           # 询问你的需求
```

## 模式

---

### 精益画布模式 (Lean Canvas Mode)

最适用于：早期想法、初创公司、新产品线。

应用 **lean-canvas** 技能生成完整的精益画布：

```
## 精益画布：[产品名称]

| 问题 (前 3 个) | 解决方案 | 独特价值主张 |
|-----------------|----------|------------------------|
| 1. [问题]       | [针对每个问题的解决方案] | [单一清晰的信息] |
| 2. [问题]       |          |                        |
| 3. [问题]       |          |                        |

| 关键指标 | 竞争优势 (护城河) |
|------------|-----------------|
| [你衡量什么] | [什么是无法被复制的] |

| 渠道 | 客户细分 |
|---------|------------------|
| [你如何触达他们] | [目标人群，优先考虑早期采用者] |

| 成本结构 | 收入来源 |
|---------------|----------------|
| [固定成本 + 可变成本] | [你如何赚钱] |

### 最具风险的假设
[为了使此模式运作，哪些必须是真的 —— 按风险排序]

### 待运行的实验
[如何以低成本验证最具风险的假设]
```

### 商业模式画布模式 (Full Business Model Canvas Mode)

最适用于：成熟产品、战略规划、投资者材料。

应用 **business-model** 技能生成所有 9 个构建块：

```
## 商业模式画布：[产品名称]

| 核心伙伴 | 关键活动 | 价值主张 | 客户关系 | 客户细分 |
|-------------|---------------|-------------------|----------------------|------------------|
| [谁在帮助你] | [核心行动] | [客户为什么选择你] | [你如何互动] | [你为谁服务] |

| 核心资源 | | 渠道 | |
|-------------|---|---------|---|
| [你需要什么] | | [你如何交付] | |

| 成本结构 | 收入来源 |
|---------------|----------------|
| [你的成本] | [你的收入] |

### 分析
[此模式的优势和劣势]
[各部分如何相互强化]
[脆弱点和依赖关系]
```

### 初创公司画布模式 (Startup Canvas Mode)

最适用于：需要将战略和商业模式整合在一个工件中的新产品和初创公司。对于新产品，推荐优先于精益画布和 BMC 使用。

应用 **startup-canvas** 技能生成包含 9 个战略板块 + 商业模式的初创公司画布：

```
## 初创公司画布：[产品名称]

### 第一部分：产品战略

| 愿景 | 市场细分 | 相对成本 |
|--------|----------------|---------------|
| [鼓舞人心的原因] | [JTBD，首个细分市场] | [低成本 vs 独特价值] |

| 价值主张 | 权衡 (取舍) | 关键指标 |
|------------------|-----------|------------|
| [之前是什么样 → 如何做 → 之后是什么样 → 替代方案] | [你不会做什么] | [北极星指标 + OMTM] |

| 增长 | 能力 | 不能/不会 (护城河) |
|--------|-------------|------------|
| [PLG vs 销售驱动，渠道] | [自建 vs 合作伙伴] | [为什么竞争对手无法复制] |

### 第二部分：商业模式

| 成本结构 | 收入来源 |
|---------------|----------------|
| [固定 + 可变，如何规模化] | [定价模型，每个渠道的收入] |

### 战略一致性检查
[所有元素是否相互强化？]

### 最具风险的假设
[什么是必须是真的 —— 以及如何测试它]
```

### 价值主张模式 (Value Proposition Mode)

最适用于：优化文案、理解用户价值、产品市场匹配 (PMF) 分析。

应用 **value-proposition** 技能生成基于 JTBD 框架的价值主张：

```
## 价值主张：[产品名称]

### 针对 [细分市场]：
1. **谁 (Who)**：[目标用户画像和特征]
2. **为什么 (Why)**：[他们试图完成的工作 (Job to be done)]
3. **之前 (What Before)**：[他们目前痛苦的现状]
4. **如何做 (How)**：[你的解决方案路径]
5. **之后 (What After)**：[他们改善后的现状]
6. **替代方案 (Alternatives)**：[如果没有你他们会用什么，以及为什么你更好]

### 价值主张陈述
[一句话总结：对于 [人群] 来说，他们 [有某种需求]，[产品名称] 是一个 [类别]，能够提供 [益处]。不同于 [替代方案]，我们 [差异化优势]。]
```

### 全部模式 (All Mode)

运行所有四个框架，并添加一个综合部分，比较各框架的见解。

## 工作流程 (所有模式)

### 第 1 步：收集背景信息

询问：
- 产品或业务想法是什么？
- 处于什么阶段？（想法阶段、已验证、规模化）
- 是否有现有的商业模式需要优化？
- 目标客户是谁？

### 第 2 步：生成所选框架

应用上述相关的技能。

### 第 3 步：保存并迭代

保存为 Markdown。提供：
- "是否需要我使用 SWOT 或 PESTLE 分析来 **对该模型进行压力测试**？"
- "我是否应该为这些收入流 **设计定价策略**？"
- "是否需要我围绕该模型 **构建战略画布**？"
- "我是否应该 **确定滩头阵地细分市场 (Beachhead Segment)**？"

## 备注

- **初创公司画布 (Startup Canvas)** 是新产品的推荐起点 —— 它将战略与商业模式分开，并涵盖了 BMC 和精益画布遗漏的内容（愿景、权衡、指标、不能/不会）
- **精益画布 (Lean Canvas)** 最适合快速迭代和假设测试 —— 不要过度思考，但要意识到它将战略和商业模式混合在一个工件中
- **BMC** 更适合需要阐明各部分如何连接的成熟业务，但缺乏战略部分（愿景、权衡、指标）
- **价值主张 (Value Proposition)** 是进行产品市场匹配对话最犀利的工具
- 在 "全部 (all)" 模式中，突出框架达成一致的地方（强信号）以及它们分歧的地方（需要调查）
