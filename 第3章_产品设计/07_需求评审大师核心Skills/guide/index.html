pm-review / 需求评审大师 · 使用指南

用 AI 预演评审会，
提前发现需求疏漏

AI 模拟前端、后端、设计、测试、业务方、老板 6 个角色，在评审会前预判所有可能的提问，并给出专业应对话术。

01为什么需要需求评审大师

需求评审会上被问到哑口无言？你的 PRD 可能已经写得很好了——只是缺了 6 个视角的提前审视。

没有预审

评审会上被连续追问 → 现场找答案 → 讨论跑偏 → 评审超时 → 需求被打回重写

散会后发现："这些问题我明明可以提前想到"

使用 pm-review

提前暴露问题 → 补充到 PRD 里 → 评审会聚焦范围确认 → 各角色看到自己的关切已被覆盖

评审结果："想得挺全面的，没什么大问题，过"

02AI 模拟的 6 个角色

每个角色从自己的专业领域出发提问，不跨角色、不笼统、分严重度。

🖥️

前端开发

关注交互实现、组件复用、状态管理、性能。会追问："加载态和错误态设计了吗？""这个组件需要复用吗？"

交互 · 状态 · 性能
🗄️

后端开发

关注数据模型、API 设计、并发安全。会追问："两个用户同时操作一条数据怎么处理？""数据量大时需要分页吗？"

数据 · 接口 · 安全
🎨

设计师

关注视觉一致性、交互规范、可用性。会追问："空状态有没有用户引导？""极端文本长度怎么处理？"

一致性 · 交互 · 可用性
🧪

测试 / QA

关注边界条件、异常路径、兼容性。会追问："最可能出 bug 的地方在哪？""弱网环境测试过吗？"

边界 · 异常 · 兼容
📊

业务方 / 运营

关注业务逻辑、用户价值、运营效率。会追问："这个真能解决用户痛点吗？""上线后怎么衡量效果？"

价值 · 逻辑 · 效果
👔

老板 / 管理者

关注战略对齐、ROI、时间资源。会追问："为什么现在做？""预计投入多少？""不做会怎样？"

战略 · 资源 · 风险
每个角色至少提出 3 个具体问题，关联到具体功能点。问题分三级：🔴 阻塞（不解决无法开发）/ 🟡 重要（影响质量）/ 🟢 建议（锦上添花）

03使用流程

从输入到输出，4 步完成一次完整的模拟评审。

输入 PRD 或原型 → AI 解析内容 → 6 角色模拟提问 → 生成评审报告

  1. 启动 Skill

    在 Claude Code 中输入 /pm-review，然后把 PRD 文本或 HTML 原型粘贴进来。支持三种输入：纯 PRD、HTML 原型、PRD + 原型混合。

  2. AI 解析需求内容

    自动提取功能点、核心流程、数据模型。如果是 HTML 原型，还会检查每个页面的加载态、空状态、错误态、成功态四态覆盖。

  3. 6 角色模拟评审

    每个角色从自己的领域出发，生成 3-5 个具体问题。问题分为 🔴阻塞 / 🟡重要 / 🟢建议 三级。相同问题自动合并去重。

  4. 生成应对方案和报告

    每个阻塞问题配套：应对方案 + 追问预判 + 兜底话术。最终输出 .md 文件，可选额外生成 .docx 和 .html。

04支持的输入类型

不只是 PRD 文本，HTML 原型同样可以评审。

输入类型识别方式特别检查项
PRD / 需求文档 文本描述、功能列表、用户故事 功能完整性、逻辑一致性、角色权限
HTML 原型 .html 文件或 HTML 代码块 四态覆盖（加载/空/错误/成功）、组件复用、交互状态
PRD + 原型混合 同时包含文档和原型 交叉验证：原型是否实现了 PRD 描述的所有功能
HTML 原型输入时，Skill 会自动提取页面结构、交互元素、状态覆盖情况，并生成专门的状态覆盖检查表。

05输出物一览

一份评审报告包含以下核心模块。

模块内容作用
评审概览仪表盘问题总数、严重度分布、应对覆盖率一眼看清评审健康度
角色评审详情每个角色提出的问题 + 应对方案 + 追问预判 + 兜底话术评审会的弹药库
应对话术库通用应对框架 + 本产品场景专用话术被突袭时不慌
改进清单按优先级排列的改进项 + 关联问题 + 操作建议评审后的行动指南
状态覆盖检查（原型输入）每个页面的四种状态检查结果原型完整度体检
完整示例见同级目录下的 需求评审报告-示例.html——以"订单管理系统（OMS）"为案例，22 个问题、5 个阻塞项、10 条改进建议。支持按角色/严重度筛选，可展开查看每个问题的完整应对方案。

06安装与配置

一行命令安装，30 秒即可开始使用。

# 安装 Skill
Copy-Item skills\pm-review\SKILL.md "$env:USERPROFILE\.claude\skills\pm-review\SKILL.md"

# 安装 docx 生成依赖（可选，需要 Word 输出时）
pip install python-docx

安装后重启 Claude Code，输入 /pm-review 即可触发。Skill 会自动识别输入类型——粘贴 PRD 文本、HTML 原型都可以。
