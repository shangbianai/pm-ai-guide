# /transform-roadmap -- 以结果为导向的路线图

将计划的功能列表或以产出物为中心的路线图，重写为以产出结果 (Outcome-focused) 为中心的路线图，强调 "为什么" 而不是 "是什么"。

## 调用示例

```
/transform-roadmap [粘贴你的功能列表或路线图]
/transform-roadmap [上传路线图文档、电子表格或截图]
```

## 工作流程

### 第 1 步：接收当前路线图

接受任何格式：
- 功能列表或待办事项
- 路线图文档（Now/Next/Later 框架、季度计划、时间线）
- 电子表格或甘特图导出文件
- 路线图工具的截图

解析每个项目以提取：功能名称、描述、目标日期/时间范围以及任何背景信息。

### 第 2 步：理解战略背景

询问：
- 该时期的产品目标或 OKR 是什么？
- 该路线图的受众是谁？（高管、工程团队、客户、董事会）
- 你偏好哪种格式？（Now/Next/Later、季度、时间线）

### 第 3 步：转化每个项目

应用 **outcome-roadmap** 技能：

针对路线图上的每个功能/产出物：
1. 识别它试图实现的 **用户或业务结果 (Outcome)**
2. 重写为结果陈述："[动词] [指标/体验] 针对 [用户群体]"
3. 将服务于同一结果的功能合并到一个倡议 (Initiative) 下
4. 为每个结果添加成功指标

**转化示例 (之前 → 之后):**
- "构建 SSO 集成" → "减少企业客户入职摩擦 —— 目标：企业账号的首次价值达成时间缩短 50%"
- "重构仪表盘" → "帮助核心用户更快发现洞察 —— 目标：洞察获取时间减少 30%"
- "添加 CSV 导出功能" → "支持团队在产品外分享数据 —— 目标：报告分享率提高 25%"

### 第 4 步：生成转化后的路线图

```
## 以结果为导向的路线图: [产品名称] — [周期]

**战略主题**: [2-3 个高层级主题]

### Now (当前季度)
**主题: [战略主题]**
| 产出结果 (Outcome) | 成功指标 | 关键倡议 | 状态 |
|---------|---------------|----------------|--------|

### Next (下个季度)
**主题: [战略主题]**
| 产出结果 (Outcome) | 成功指标 | 关键倡议 | 置信度 |
|---------|---------------|----------------|------------|

### Later (未来)
**主题: [战略主题]**
| 产出结果 (Outcome) | 成功指标 | 关键倡议 | 依赖项 |
|---------|---------------|----------------|-------------|

### 转化备注
| 原始功能 | 转化后的结果 | 为什么要这样表述 |
|-----------------|--------------------|-----------------|

### 变更总结
[路线图叙事逻辑变化的简要说明]
```

保存为 markdown 文件。

### 第 5 步：评审

提供建议：
- "需要我为每个结果 **添加 OKR 对齐说明** 吗？"
- "我是否应该为该路线图 **起草一份利益相关者演示文稿**？"
- "需要我为 'Now' 阶段的项目 **识别风险** 吗？"

## 注意事项

- 产出结果 (Outcomes) 应该是可衡量的，并且有明确的 "完成" 状态
- 多个功能可以映射到一个结果 —— 这是有意为之，而非 Bug
- 如果某个产出物 (Output) 无法清晰服务于任何结果，请将其标记出来，供用户评估其合理性或降低优先级
- 受众很重要：给高管看的路线图应仅包含结果，给工程团队看的路线图可以在每个结果下包含交付物
- "Later" 阶段的项目应保持较低的详细程度 —— 只有结果，没有承诺的解决方案
