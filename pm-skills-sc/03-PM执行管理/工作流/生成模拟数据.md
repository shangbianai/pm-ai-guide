# /generate-data -- 测试数据生成器

为开发、测试、演示或原型设计创建真实的模拟数据集。以你首选的格式输出可直接使用的文件。

## 调用示例

```
/generate-data 1000 个用户，包含姓名、电子邮件、订阅层级、注册日期和活跃度评分
/generate-data 电子商务订单数据集：产品、客户、时间戳、金额
/generate-data 符合此架构的样本数据：[粘贴表定义]
```

## 工作流程

### 第 1 步：定义数据集

明确以下内容：
- **哪些实体？**（用户、订单、事件、产品等）
- **哪些列？**（包含数据类型和约束）
- **多少行？**
- **表之间是否有关系？**
- **是否有特定的分布要求？**（例如："80% 应该是免费计划用户"）
- **是否有现实约束？**（电子邮件必须唯一，日期必须按时间顺序排列）

### 第 2 步：生成数据

应用 **dummy-dataset** 技能：

- 创建一个生成数据集的 Python 脚本
- 使用看起来真实的数据（不是随机字符串）：正确的姓名、有效的电子邮件格式、看起来真实的日期
- 遵守约束：唯一 ID、外键关系、时间顺序
- 应用指定的分布比例
- 执行脚本并生成输出文件

### 第 3 步：交付

以请求的格式输出（或询问）：
- **CSV**: 最常用，兼容性最好
- **JSON**: 用于 API 测试或前端开发
- **SQL INSERT**: 用于填充测试数据库
- **Python 脚本**: 用于可重复生成（用户可以调整并重新运行）

```
## 生成的数据集：[描述]

**行数**: [数量]
**列名**: [列表]
**格式**: [CSV / JSON / SQL / Python]

### 架构 (Schema)
| 列名 | 类型 | 约束 | 分布 |
|--------|------|-----------|-------------|

### 样本（前 5 行）
[数据预览]

### 文件
- [数据文件]
- [生成器脚本，如果适用]
```

将数据文件和生成器脚本保存到用户的工作区。

### 第 4 步：后续建议

- "需要我 **添加更多列** 或 **增加数据集大小** 吗？"
- "我是否应该 **创建相关联的表**（例如：这些用户的订单）？"
- "需要我 **编写使用这些数据的测试场景** 吗？"
- "我是否应该 **创建 SQL 查询** 来分析这个数据集？"

## 注意事项

- 始终提供生成器脚本，以便用户可以使用不同的参数重新生成
- 对于演示数据集，让数据能够 "讲故事"（例如：季节性趋势、留存问题、核心用户群体）
- 遵守现实中的基数 (Cardinality)：1000 个用户不应该有 1000 个唯一的城市
- 对于财务数据，使用现实的价格分布 —— 而不是均匀随机分布
- 严禁包含真实的个人数据 —— 所有姓名、电子邮件和标识符必须是虚构的
