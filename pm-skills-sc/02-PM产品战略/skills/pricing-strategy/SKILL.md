## 定价战略 (Pricing Strategy)

设计一个基于价值交付、竞争定位和付费意愿的定价战略。

### 上下文

您正在为 **$ARGUMENTS** 制定定价战略。

如果用户提供了文件（竞争对手定价、调查数据、财务模型或使用数据），请先阅读。如果需要，使用 Web 搜索来调研竞争对手的定价。

### 指导说明

1. **理解交付的价值**：
   - 核心价值主张是什么？
   - 客户的替代方案是什么（及其成本）？
   - 产品交付了哪些可量化的结果？（节省的时间、获得的收入、降低的成本）
   - 基于该价值，客户的付费意愿是多少？

2. **评估定价模型** —— 推荐最合适的模型：

   | 模型 | 适用场景 | 示例 |
   |---|---|---|
   | **固定费率 (Flat-rate)** | 简单的产品，成本可预测 | Basecamp (每月 $99 固定) |
   | **按席位 (Per-seat)** | 协作工具，团队产品 | Slack, Figma |
   | **基于用量 (Usage-based)** | 基础设施，API 产品 | AWS, Twilio |
   | **分层定价 (Tiered)** | 具有明显用户细分的产品 | 大多数 SaaS (免费/专业/企业) |
   | **免费增值 (Freemium)** | 具有病毒式/网络效应的产品 | Spotify, Notion |
   | **免费增值 + 用量** | 平台类产品 | Vercel, OpenAI API |
   | **基于价值 (Value-based)** | 高影响力的企业工具 | Salesforce, Palantir |

3. **分析竞争定价**：
   - 映射竞争对手的定价层级及包含的内容
   - 识别您的产品定位（高端、中端、低端）
   - 寻找定价空白或机会
   - 注意任何行业定价惯例

4. **设计定价结构**：
   - **层级 (Tiers)**：定义 2-4 个具有清晰差异化的层级
   - **功能门控 (Feature gating)**：哪些功能放在哪个层级？（使用价值指标，而非随意限制）
   - **价值指标 (Value metric)**：您按什么单位收费？（用户数、事件数、存储空间、API 调用次数）
   - **锚点定价 (Anchor pricing)**：设置最受欢迎的层级，使其看起来像是显而易见的选择
   - **年度折扣**：通常比月度定价优惠 15-20%

5. **估算价格敏感度**：
   - Van Westendorp 价格敏感度测量 (如果调查数据可用)：
     - 太便宜 → 质量担忧
     - 便宜 → 划算
     - 贵 → 开始犹豫
     - 太贵 → 不会购买
   - 或者，基于竞争对手定价和交付价值进行估算

6. **规划定价实验**：
   - A/B 测试定价页面（不同价格点、层级名称、功能组合）
   - 创始人主导的销售对话以测试付费意愿
   - 使用不同价格锚点的落地页测试
   - 按价格点进行的转化率队列分析 (Cohort analysis)

7. **输出定价建议**：
   ```
   推荐模型：[模型类型]
   价值指标：[收费单位]

   | 层级 | 价格 | 目标细分群体 | 关键功能 | 定位 |
   |---|---|---|---|---|

   关键假设：
   - [假设] → [如何测试]

   风险：
   - [风险] → [缓解措施]
   ```

逐步思考。保存为 markdown。标记任何在发布前需要验证的假设。
