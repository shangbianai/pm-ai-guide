# 设计指南

幻灯片的详细设计原则。

## 受众指南

设计决策应根据目标受众进行调整。使用 `--audience` 进行设置。

| 受众 | 内容密度 | 视觉风格 | 术语使用 | 幻灯片数量 |
|----------|-----------------|--------------|-------------|--------|
| `beginners` | 低 | 友好、示例丰富 | 通俗语言 | 8-15 |
| `intermediate` | 中 | 平衡、结构化 | 可适度使用专业术语 | 10-20 |
| `experts` | 高 | 数据密集、精准 | 技术术语 | 12-25 |
| `executives` | 中低 | 简洁、有冲击力 | 商业语言 | 8-12 |
| `general` | 中 | 易理解、有吸引力 | 尽量少用术语 | 10-18 |

### 受众 → 密度映射

根据受众推荐的密度等级：

| 受众 | 推荐密度 | 理由 |
|----------|-------------------|-----------|
| `executives` | minimal | 每页一个洞察，尊重时间 |
| `beginners` | minimal → balanced | 单一概念，逐步建立认知 |
| `general` | balanced | 易理解同时兼顾信息量 |
| `intermediate` | balanced | 标准信息密度 |
| `experts` | balanced → dense | 每页可承载更多数据 |

**自动密度选择**：
- 若 `--audience executives` → 默认使用 `minimal` 密度
- 若 `--audience beginners` → 默认使用 `minimal` 或 `balanced`
- 若 `--audience experts` → 允许使用 `dense` 密度
- 其他情况 → 默认使用 `balanced`

### 针对不同受众的设计原则

**初学者**：
- 每页只讲一个概念
- 用视觉隐喻替代抽象图表
- 循序渐进的递进式结构
- 充裕的留白

**专家**：
- 每页可展示多个数据点
- 精确标注的技术图表
- 假定受众具备领域知识
- 信息密集但井然有序

**高管**：
- 以洞察为先，而非罗列数据
- 每页都要回答"所以呢？"
- 内容应便于做出决策
- 结论先行（BLUF）

## 视觉层级原则

| 原则 | 说明 |
|-----------|-------------|
| 焦点 | 每页一个主导元素，优先吸引注意力 |
| 三分法则 | 将关键元素放置在网格交叉点上 |
| Z 字形视线 | 引导视线：左上 → 右上 → 左下 → 右下 |
| 大小对比 | 标题字号为正文的 2-3 倍 |
| 呼吸空间 | 距离所有边缘至少保留 10% 的边距 |

## 内容密度

完整密度规格参见 `references/dimensions/density.md`。

| 等级 | 说明 | 适用场景 |
|-------|-------------|----------|
| 高 | 多个数据点、详细图表、密集文字 | 专家受众、技术评审 |
| 中 | 要点配合支撑细节 | 通用商务、混合受众 |
| 低 | 一个核心观点、大视觉元素、极少文字 | 初学者、主题演讲、情感传达 |

**高密度原则**（麦肯锡风格）：
- 每个元素都要有存在的价值
- 数据比装饰更有说服力
- 注释解释洞察，而非复述数据
- 留白是策略，不是填充

**按幻灯片类型的密度建议**：
| 幻灯片类型 | 推荐密度 |
|------------|-------------------|
| 封面/标题页 | minimal |
| 议程/概览 | balanced |
| 内容/分析 | balanced 或 dense |
| 数据/指标 | dense |
| 引言/亮点 | minimal |
| 总结/要点 | balanced |

## 配色选择

完整情绪规格参见 `references/dimensions/mood.md`。

**内容优先方法**：
1. 分析内容主题、情绪调性和所属行业
2. 考虑目标受众的期望
3. 让配色与主题匹配
4. 确保足够对比度以保证可读性

**快速配色指南**：
| 内容类型 | 推荐情绪 |
|--------------|-----------------|
| 技术/架构 | cool |
| 教育/友好 | warm |
| 企业/专业 | professional |
| 创意/艺术 | vibrant |
| 科学/医疗 | cool 或 neutral |
| 娱乐/游戏 | dark 或 vibrant |

## 排版原则

完整排版规格参见 `references/dimensions/typography.md`。

| 元素 | 处理方式 |
|---------|-----------|
| 标题 | 加粗、字号为正文 2-3 倍、叙述风格 |
| 正文 | 常规字重、可读字号 |
| 说明文字 | 较小字号、较细字重 |
| 数据标签 | 技术内容使用等宽字体 |
| 强调 | 使用加粗或颜色，不用下划线 |

## 字体推荐

**英文字体**：
| 字体 | 风格 | 最适用场景 |
|------|-------|----------|
| Liter | 无衬线、几何风格 | 现代、简洁、技术 |
| HedvigLettersSans | 无衬线、独特个性 | 品牌导向、创意 |
| Oranienbaum | 高对比衬线 | 优雅、经典 |
| SortsMillGoudy | 经典衬线 | 传统、易读 |
| Coda | 圆润无衬线 | 友好、亲切 |

**中文字体**：
| 字体 | 风格 | 最适用场景 |
|------|-------|----------|
| MiSans | 现代无衬线 | 简洁、通用、屏幕优化 |
| Noto Sans SC | 中性无衬线 | 标准、多语言 |
| siyuanSongti | 精致宋体 | 优雅、编辑排版 |
| alimamashuheiti | 几何无衬线 | 商业、结构化 |
| LXGW Bright | 宋楷混合 | 温暖、易读 |

**多语言搭配**：
| 使用场景 | 英文 | 中文 |
|----------|---------|---------|
| 技术 | Liter | MiSans |
| 编辑排版 | Oranienbaum | siyuanSongti |
| 友好风格 | Coda | LXGW Bright |
| 企业 | HedvigLettersSans | alimamashuheiti |

## 视觉元素参考

完整纹理规格参见 `references/dimensions/texture.md`。

### 背景处理

| 处理方式 | 说明 | 最适用场景 |
|-----------|-------------|----------|
| 纯色 | 单一背景色 | 简洁、极简 |
| 分割背景 | 两种颜色，对角线或垂直分割 | 对比、分区 |
| 渐变 | 柔和的垂直或对角渐变 | 现代、动感 |
| 纹理 | 图案或纹理叠加 | 个性、风格化 |

### 排版处理

| 处理方式 | 说明 | 最适用场景 |
|-----------|-------------|----------|
| 大小对比 | 标题与正文 3-4 倍字号差 | 冲击力、层级感 |
| 全大写标题 | 大写字母配合字间距 | 权威感、结构感 |
| 等宽数据 | 等宽字体用于数字/代码 | 技术、精确性 |
| 手绘风格 | 自然、不完美的字形 | 友好、亲切 |

### 几何装饰

| 元素 | 说明 | 最适用场景 |
|---------|-------------|----------|
| 对角分割线 | 倾斜的区域分隔线 | 活力、动感 |
| 直角边框 | L 形边框 | 聚焦、框定 |
| 圆形/六边形 | 用形状框住图片 | 现代、科技感 |
| 下划线装饰 | 标题下方的粗线条 | 强调、层级感 |

## 一致性要求

| 元素 | 指导原则 |
|---------|-----------|
| 间距 | 全文保持一致的边距和内边距 |
| 配色 | 每页最多 3-4 种颜色，整套幻灯片配色保持一致 |
| 排版 | 相同内容类型使用相同的字体族和字号 |
| 视觉语言 | 重复使用相同的图案、形状和处理方式 |

## 维度组合指南

组合各维度时，请考虑兼容性：

| 受众 | 推荐维度组合 |
|----------|----------------------|
| 高管 | clean + neutral + geometric + minimal |
| 初学者 | organic + warm + humanist + minimal |
| 大众 | 任意纹理 + 任意情绪 + humanist/geometric + balanced |
| 专家 | grid/clean + cool + technical + balanced/dense |

| 内容类型 | 推荐维度组合 |
|--------------|----------------------|
| 教程 | organic + warm + handwritten + balanced |
| 技术 | grid + cool + technical + balanced |
| 商务 | clean + professional + geometric + balanced |
| 创意 | organic + vibrant + humanist + balanced |
| 数据密集 | clean + cool + technical + dense |
