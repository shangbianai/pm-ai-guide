# /north-star -- 北极星指标定义 (North Star Metric Definition)

识别最能体现产品价值的单一指标，以及驱动该指标的输入指标。对你的业务游戏进行分类，并根据经过验证的标准进行验证。

## 调用示例

```
/north-star 面向团队协作的 B2B SaaS
/north-star 通过订阅获利的消费级健身应用
/north-star 帮助我们修正北极星指标 —— 我们在追踪 DAU 但感觉不对
```

## 工作流程

### 第 1 步：了解产品

询问：
- 产品是什么？它为用户提供什么价值？
- 商业模式是什么？（订阅、交易、广告、平台/市场）
- 目前追踪的指标（如果有）
- 为什么现在需要定义？（新产品、现有指标感觉不对、团队对齐）

### 第 2 步：对业务游戏进行分类

应用 **north-star-metric** 技能：

识别产品属于哪种业务游戏：
- **注意力 (Attention)**：通过用户时长/参与度获利（媒体、社交、广告支持）
- **交易 (Transaction)**：通过购买获利（电子商务、平台/市场）
- **生产力 (Productivity)**：通过效率提升获利（SaaS、工具、B2B）

业务游戏决定了什么样的北极星指标才有意义。

### 第 3 步：定义北极星指标

- 提出 2-3 个北极星指标候选方案
- 根据 7 个标准验证每个方案：
  1. 表述了传递给客户的价值
  2. 是收入的领先指标
  3. 可衡量且可追踪
  4. 全团队易于理解
  5. 可操作（团队可以影响它）
  6. 不是虚荣指标
  7. 不会在不传递真实价值的情况下被操纵
- 推荐最强的候选方案并说明理由

### 第 4 步：定义输入指标

为选定的北极星指标识别 3-5 个输入指标：
- 每个输入指标都应该是直接驱动北极星指标的杠杆
- 每个指标都应由特定团队负责
- 输入指标应能 MECE（相互独立，完全穷尽）地解释北极星指标的变化

### 第 5 步：生成指标框架

```
## 北极星框架：[产品名称]

**业务游戏**: [注意力 / 交易 / 生产力]

### 北极星指标 (North Star Metric)
**指标**: [准确名称]
**定义**: [公式或衡量方法]
**为什么选择此指标**: [解释价值、引导收入、可操作性]
**当前值**: [如果已知]
**目标**: [目标值]

### 验证
| 标准 | 是否通过？ | 备注 |
|------|------------|------|
| 表述价值 | [是/否] | [解释] |
| 领先指标 | [是/否] | [解释] |
| 可衡量 | [是/否] | [解释] |
| 易于理解 | [是/否] | [解释] |
| 可操作 | [是/否] | [解释] |
| 非虚荣指标 | [是/否] | [解释] |
| 不可操纵 | [是/否] | [解释] |

### 输入指标 (Input Metrics)
| 输入指标 | 如何驱动北极星 | 负责人 | 当前值 | 目标值 |
|----------|----------------|--------|--------|--------|

### 指标星座 (Metrics Constellation)
[展示 北极星 → 输入指标 → 团队行动 的视觉树]

| 指标 | 防止什么问题 |
|------|--------------|

### 避开的误区
[为什么我们没有选择 DAU、收入或其他常见但有缺陷的指标]
```
