# 演示文稿分析框架

深度内容分析，用于高效创建幻灯片。

## 1. 信息层级

在设计幻灯片之前，先识别核心信息结构。

### 核心信息（一句话）
- 最重要的核心结论是什么？
- 如果听众只能记住一件事，应该是什么？
- 能否用 ≤15 个字概括？

### 支撑要点（最多 3-5 个）
- 有哪些证据支撑核心信息？
- 必须涵盖哪些子话题？
- 按听众相关性排序，而非按原始素材顺序

### 行动号召（Call-to-Action）
- 听众看完后应该做什么？
- 是否清晰、具体、可执行？
- 它出现在哪里（幻灯片位置）？

## 2. 听众决策矩阵

| 问题 | 分析 |
|------|------|
| 主要听众是谁？ | [角色、专业水平、与主题的关系] |
| 他们目前相信什么？ | [已有知识、假设、偏见] |
| 我们希望他们做出什么决定？ | [具体行动或结论] |
| 存在哪些障碍？ | [反对意见、顾虑、信息缺失] |
| 什么证据能说服他们？ | [数据类型、可信来源、情感触发点] |

### 听众适配

| 听众类型 | 内容重点 | 视觉处理 |
|----------|----------|----------|
| 高管层 | 成果、ROI、战略影响 | 高层次、简洁、数据亮点 |
| 技术人员 | 架构、实现方案、技术规格 | 详细图表、代码、原理图 |
| 普通受众 | 收益、故事、共鸣感 | 视觉隐喻、简洁图表 |
| 投资人 | 市场规模、发展势头、团队 | 增长曲线、里程碑、对比 |
| 学习者 | 分步骤讲解、案例、练习 | 渐进式展示、实操练习 |

## 3. 视觉化机会图谱

识别哪些内容适合可视化呈现。

### 内容到视觉的映射

| 内容类型 | 视觉处理 | 示例 |
|----------|----------|------|
| 对比 | 左右并排、前后对比 | 功能对比表 |
| 流程 | 流程图、编号步骤 | 工作流示意图 |
| 层级 | 组织架构图、金字塔、树形图 | 组织结构 |
| 时间线 | 水平/垂直时间轴 | 项目里程碑 |
| 统计数据 | 图表、高亮数字 | 带上下文的关键指标 |
| 概念 | 图标、隐喻、插图 | 抽象想法可视化 |
| 关系 | 韦恩图、网络图 | 生态系统或依赖关系 |
| 列表 | 结构化网格、图标行 | 带图标的功能要点 |

### 视觉化优先级

对每段内容进行评级：
- **必须可视化**：复杂数据、核心差异化优势、令人难忘的关键时刻
- **应该可视化**：支撑证据、次要要点
- **仅文字**：简单陈述、过渡、次要细节

## 4. 演示流程

为影响力和记忆留存设计结构。

### 开场（前 2-3 张幻灯片）

| 要素 | 目的 |
|------|------|
| 抓手（Hook） | 抓住注意力（令人惊讶的数据、提问、故事） |
| 背景 | 为什么此刻很重要 |
| 预告 | 听众将学到/获得什么 |

### 中段（内容幻灯片）

| 模式 | 适用场景 |
|------|----------|
| 问题 → 解决方案 | 介绍新产品/新想法 |
| 现状 → 冲突 → 解决（SCR） | 复杂商业案例 |
| 是什么 → 为什么 → 怎么做 | 教育类内容 |
| 过去 → 现在 → 未来 | 变革故事 |
| 论点 → 证据 → 推论 | 数据驱动的论证 |

### 结尾（最后 2-3 张幻灯片）

| 要素 | 目的 |
|------|------|
| 总结 | 回扣核心信息 |
| 行动号召 | 明确下一步 |
| 令人难忘的收尾 | 引发共鸣的金句、图片或陈述 |

### 过渡

- 每张幻灯片都应回答：「接下来是什么？」
- 在各章节之间使用叙事性衔接
- 构建逻辑递进，而非话题跳跃

## 5. 内容适配

决定保留、转化还是省略。

### 保留（高价值）
- 核心论点和证据
- 独到见解或数据
- 与听众相关的案例
- 令人难忘的金句或统计数据

### 简化（中等价值）
- 技术细节 → 视觉摘要
- 冗长解释 → 层级要点
- 多个案例 → 最好的 1-2 个
- 背景信息 → 简要铺垫

### 可视化（转化）
- 数据表格 → 图表或高亮数字
- 流程描述 → 流程图
- 文字对比 → 左右并排视觉图
- 抽象概念 → 具体隐喻

### 省略（低价值）
- 边缘信息
- 重复案例
- 过多的限定条件
- 听众已知的背景知识

## 6. 分析检查清单

在创建大纲之前，确认：

### 信息清晰度
- [ ] 核心信息已用一句话概括
- [ ] 已识别 3-5 个支撑要点
- [ ] 已定义行动号召

### 听众匹配
- [ ] 已识别主要听众
- [ ] 已梳理听众已有认知
- [ ] 期望的决策已明确
- [ ] 证据与听众需求匹配

### 视觉规划
- [ ] 已识别关键可视化内容
- [ ] 已选择图表/图示类型
- [ ] 已分配视觉化优先级

### 流程设计
- [ ] 已设计开场抓手
- [ ] 已选择中段模式
- [ ] 已规划结尾方式
- [ ] 已考虑过渡衔接

### 内容决策
- [ ] 已完成保留/简化/可视化/省略的判断
- [ ] 已充分处理原始素材
- [ ] 无重要内容遗漏
