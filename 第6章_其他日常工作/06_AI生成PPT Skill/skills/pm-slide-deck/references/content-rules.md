# 内容与风格规则

幻灯片内容质量与风格一致性指南。

## 内容规则

### 1. 尊重读者注意力
- 每页幻灯片只传达一个核心观点
- 移除冗余信息
- 优先保证清晰度，而非面面俱到

### 2. 数据可溯源
- 所有统计数据必须标注来源
- 在包含数据的幻灯片上直接引用来源
- 使用具体数字，而非模糊表述

### 3. 自包含的提示词
- 所有细节必须写在图片提示词中
- 禁止外部引用（例如「参考第 2 页」）
- 明确包含所有颜色、布局和内容

### 4. 禁止占位符
- 每个元素必须完整指定
- 禁止出现「[在此填入数据]」或「TBD」
- 所有文本内容在生成前必须确定

## 风格规则

### 1. 叙事性标题
标题应讲述故事，而非仅仅标注内容。

| 反面示例 | 正面示例 |
|-----|------|
| 「关键数据」 | 「使用量在 6 个月内翻倍」 |
| 「我们的方案」 | 「一个平台替代五个工具」 |
| 「优势」 | 「团队每周节省 10 小时」 |

### 2. 避免AI陈词滥调
移除以下模式：
- 「深入探讨」「探索」「旅程」
- 「让我们看看」「我来展示」
- 「令人兴奋的」「惊人的」「革命性的」
- 「总之」「总结一下」

### 3. 有意义的封底
不要只写「谢谢」或「有问题吗？」

应包含以下之一：
- 明确的行动号召
- 令人印象深刻的核心要点
- 引发思考的结束语
- 有明确用途的联系方式

### 4. 统一的视觉语言
整套幻灯片中保持：
- 统一的图标风格
- 统一的色彩使用模式
- 统一的布局网格系统
- 统一的字体层级

## 幻灯片结构

| 位置 | 类型 | 用途 |
|----------|------|---------|
| 1 | 封面 | 标题、视觉吸引点、主题引入 |
| 2 到 N-1 | 内容 | 关键要点、数据、说明 |
| N | 封底 | 总结、行动号召、令人印象深刻的收尾 |

## 关键规格

| 规格项 | 值 |
|---------------|-------|
| 宽高比 | 16:9（横向） |
| 幻灯片数量 | 根据内容动态确定 |
| 必需页面 | 至少包含封面 + 封底 |
| 页脚 | 无（无页码、无 logo） |
| 语言优先级 | `--lang` → 源语言 → 询问用户 |
| 语气 | 直接、自信（避免AI式用语） |

## 风格速查

| 风格 | 视觉概述 |
|-------|----------------|
| `sketch-notes` | 手绘风格，暖色米白底，概念图标 |
| `blueprint` | 技术蓝图，网格纹理，蓝色调 |
| `bold-editorial` | 高对比度，深色背景，杂志冲击感 |
| `vector-illustration` | 扁平矢量，黑色轮廓，复古配色 |
| `minimal` | 极致留白，单一强调色，禅意风格 |
| `storytelling` | 全出血图片，电影感，情绪化表达 |
| `warm` | 柔和渐变，圆润形状，健康调色盘 |
| `notion` | 仪表盘美学，简洁数据可视化，SaaS 风格 |
| `corporate` | 海军蓝/金色，结构化布局，商务精致感 |
| `playful` | 鲜艳珊瑚/青绿/黄色，动感活力 |

完整风格规格：`references/styles/<style>.md`
