# /review-resume -- PM 简历审查

根据产品管理最佳实践对简历进行深入审查。评估结构、影响指标、关键词优化，并提供带有示例的具体改进建议。

## 调用方式

```
/review-resume [粘贴简历文本]
/review-resume [上传简历 PDF 或 DOCX]
```

## 工作流程

### 第 1 步：接收简历

接收粘贴的文本、上传的 PDF 或 DOCX 文件。解析完整内容。

### 第 2 步：根据 10 项最佳实践进行评估

应用 **review-resume** 技能：

1. **影响指标 (Impact Metrics)**：成就是否量化？（收入、用户、转化率）
2. **XYZ+S 公式**：使用 [S 技能/工具] 执行 [Z]，实现了 [X]（通过 [Y] 衡量）
3. **PM 专业语言**：使用产品术语（发布、主导探索、定义策略）
4. **结构与可读性**：章节清晰、格式一致、易于扫描
5. **关键词优化**：匹配常见的 PM 职位描述关键词
6. **职业轨迹 (Story Arc)**：展示职业发展和职责范围的扩大
7. **简洁性**：初级一页，资深最多两页。不含废话。
8. **相关性**：针对 PM 角色量身定制经验，而非通用内容
9. **技术可信度**：展示与工程、数据、设计的协作
10. **领导力信号**：跨职能影响力、利益相关者管理、导师带教

### 第 3 步：生成审查报告

```
## 简历审查报告

**总分**: [X/10]
**最强项**: [哪项最佳实践]
**最大机会点**: [哪项最佳实践]

### 评分卡
| # | 最佳实践 | 分数 | 评估 |
|---|-------------|-------|-----------|
| 1 | 影响指标 | [/10] | [简要评估] |
| 2 | XYZ+S 公式 | [/10] | [简要评估] |
| ... | ... | ... | ... |

### 前 3 项改进建议

**1. [最具影响力的更改]**
- 当前: "[简历中的确切文本]"
- 建议: "[改进后的版本]"
- 原因: [理由]

**2. [第二项改进]**
[相同格式]

**3. [第三项改进]**
[相同格式]

### 逐项反馈
[针对简历每个章节的具体说明：摘要、经验、教育、技能]

### 缺失元素
[PM 简历中应该存在但目前缺失的内容]

### 待添加关键词
[简历中缺失但在典型职位描述中出现的 PM 相关关键词]
```

### 第 4 步：提供后续步骤

- “想让我针对特定职位描述**量身定制这份简历**吗？”
- “我应该使用 XYZ+S 公式**重写特定的要点**吗？”
- “想让我根据这份简历**生成一封求职信**吗？”

## 注意事项

- 保持具体且具有建设性 —— “添加指标”是没有帮助的，“将‘改进了入职流程’更改为‘通过将入职流失率降低 23%（每月 450 → 347 名用户）’”是可操作的
- PM 简历应强调结果而非产出，强调影响力而非权力
- ATS (申请人追踪系统) 优化非常重要 —— 自然地提及相关关键词
- 不同级别的 PM 有不同的期望：APM = 潜力，资深 PM = 影响力，总监及以上 = 规模
