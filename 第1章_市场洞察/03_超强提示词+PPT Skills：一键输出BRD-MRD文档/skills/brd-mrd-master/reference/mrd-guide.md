# MRD 编写方法论指南

## 什么是 MRD

MRD（Market Requirements Document，市场需求文档）是产品规划阶段的重要文档，回答的核心问题是：**"市场要什么？做什么才能赢？"**

MRD 面向的读者通常是产品团队、技术负责人、市场团队，目的是明确市场需求和产品方向，为 PRD（产品需求文档）的编写提供输入。

**MRD vs BRD 的区别：**

| 维度 | BRD | MRD |
|------|-----|-----|
| 核心问题 | 值不值得做？ | 市场要什么？ |
| 主要读者 | 高管、投资人 | 产品、技术、市场团队 |
| 关注重点 | 商业价值、ROI、资源 | 市场需求、用户痛点、竞品 |
| 输出方向 | "Go/No-Go"决策 | 产品方向和优先级 |

---

## MRD 的核心要素

### 1. 执行摘要

MRD 的执行摘要聚焦于市场机会和核心发现：

**编写原则：**
- 一句话说清市场机会
- Top 3 核心发现，每个都有数据支撑
- 明确的下一步行动建议

**好的示例：**
> 中国企业级协同办公市场2025年预计达到480亿元，年均增速18.5%，但中小企业渗透率仅35%。核心发现：1) 移动端协同是刚需但体验差距大；2) 行业垂直化是增长突破口；3) AI+协同正处于早期红利期。建议：聚焦中小企业移动场景，3个月内推出MVP。

---

### 2. 市场机会评估

#### 市场容量评估

**TAM/SAM/SOM 估算（同 BRD 指南）**

需要额外关注：
- 市场当前阶段（萌芽/成长/成熟/衰退）
- 市场集中度（CR3/CR5/HHI）
- 市场增长驱动因素

#### 增长率分析

**多维增长率分析框架：**
```
历史增长率
├── 过去 3 年 CAGR
├── 最近 1 年增长率
└── 增长趋势（加速/减速/平稳）

预测增长率
├── 乐观/中性/保守三种情景
├── 增长驱动力分析
└── 增长天花板判断
```

#### 渗透率分析

**渗透率评估的关键问题：**
- 当前市场的产品渗透率是多少？
- 不同细分市场的渗透率差异？
- 渗透率的天花板在哪里？
- 渗透率提升的主要障碍？

---

### 3. 用户需求分析

#### 用户画像构建

**画像构建的 6 个维度：**
1. 基础属性：年龄、性别、职业、收入
2. 行为特征：使用习惯、决策路径
3. 核心需求：最想解决的问题
4. 痛点：最不满意的地方
5. 触媒习惯：从哪里获取信息
6. 消费能力：付费意愿和预算

**画像质量检查：**
- ✅ 有具体场景和故事
- ✅ 有量化的行为数据
- ✅ 痛点可验证
- ❌ 只有人口统计信息
- ❌ 痛点太笼统

#### 痛点优先级矩阵

用"频率 × 严重度"评估痛点优先级：

| | 低严重度 | 高严重度 |
|---|---|---|
| **高频率** | 🟡 优化项 | 🟢 P0 必须解决 |
| **低频率** | 🔵 可忽略 | 🟡 看影响范围决定 |

**痛点描述规范：**
- ❌ "用户体验不好"
- ✅ "新用户从注册到首次使用核心功能平均需要7步、耗时3分钟，首周流失率65%"

---

### 4. 竞品分析

#### 三层竞品分析

**直接竞品**：解决相同问题的产品
- 市场份额
- 产品功能对比
- 定价策略
- 用户评价（App Store、G2、知乎等）
- 最新动态

**间接竞品**：用不同方式解决同一问题的产品
- 替代方案的本质
- 威胁程度评估
- 转化可能性

**替代方案**：用户当前不用任何产品的做法
- 手工处理方案
- 线下解决方案
- "不解决"的成本

#### SWOT 竞品对比矩阵

```
            │ 竞品A  │ 竞品B  │ 我们
────────────┼───────┼───────┼─────
功能完整度   │  ⭐⭐⭐  │  ⭐⭐  │ ⭐⭐
价格竞争力   │  ⭐⭐  │  ⭐⭐⭐ │ ⭐⭐⭐
用户体验     │  ⭐⭐⭐ │  ⭐⭐  │ ⭐⭐
技术壁垒     │  ⭐⭐  │  ⭐⭐⭐ │ ⭐⭐
品牌认知     │  ⭐⭐⭐ │  ⭐⭐  │ ⭐
```

### 5. 产品建议

#### 功能需求优先级

**RICE 评分法：**
```
RICE Score = (Reach × Impact × Confidence) / Effort

- Reach：影响多少用户（人数/次/月）
- Impact：影响程度（0.25最小化 / 0.5低 / 1中 / 2高 / 3极大）
- Confidence：确信度（50%低 / 80%中 / 100%高）
- Effort：所需人月
```

#### 用户体验建议

从以下维度给出建议：
- **信息架构**：功能组织方式
- **交互流程**：核心任务流程
- **视觉风格**：参考竞品和行业标杆
- **新手引导**：首次体验优化
- **关键场景**：高价值场景的体验设计

#### 技术方向建议

- 推荐的技术栈（简要说明原因）
- 关键技术挑战
- 第三方服务选择
- 技术债务预估

### 6. GTM（Go-To-Market）策略建议

#### 定价策略

**常用定价模型：**

| 模型 | 适用场景 | 例子 |
|------|----------|------|
| 按席位订阅 | B2B SaaS | Slack、飞书 |
| 按用量计费 | 基础设施/工具 | AWS、Twilio |
| 免费增值 | 消费级产品 | Notion、Canva |
| 阶梯定价 | 多套餐选择 | 大多数 SaaS |

**定价三步法：**
1. 确定价值锚点（用户愿意为"核心价值"付多少钱）
2. 竞品定价对标（在竞品价格带中的定位）
3. 分层套餐设计（引流版/专业版/企业版）

#### 渠道策略

**B2C 产品渠道矩阵：**

| 渠道类型 | 具体渠道 | 适用阶段 |
|----------|----------|----------|
| 应用商店 | App Store、各大应用市场 | 全阶段 |
| 社交媒体 | 微信、抖音、小红书、B站 | 增长期 |
| 搜索引擎 | SEO、SEM | 增长期 |
| 内容营销 | 公众号、知乎、博客 | 全阶段 |
| KOL/KOC | 达人合作、口碑种草 | 早期启动 |

**B2B 产品渠道矩阵：**

| 渠道类型 | 具体渠道 | 适用阶段 |
|----------|----------|----------|
| 自主销售 | 电销、面销 | 全阶段 |
| 渠道合作 | 代理商、集成商 | 规模化 |
| 内容营销 | 白皮书、行业报告 | 全阶段 |
| 行业展会 | 垂直行业展会 | 早期 |
| 客户推荐 | 老带新、案例营销 | 成熟期 |

#### 推广策略

**按阶段分配推广预算：**

```
阶段一（0-3个月）· 种子期 15%
├── 产品发布活动
├── 种子用户邀请
└── KOL 种子投放

阶段二（3-6个月）· 增长期 45%
├── 付费获客（SEM/信息流）
├── 内容营销体系搭建
└── 合作渠道开拓

阶段三（6-12个月）· 规模期 40%
├── 品牌建设
├── 大规模付费推广
└── 生态合作扩展
```

### 7. 数据支撑

#### 行业数据引用规范

**引用数据必须标注：**
- 数据来源（机构名称）
- 发布时间
- 数据口径
- 与本项目的关联性

**可信数据源：**
- 行业报告：艾瑞、QuestMobile、IDC、Gartner
- 财报数据：上市公司年报
- 政府统计：国家统计局、各部委
- 学术研究：知网、Google Scholar
- 用户调研：一手调研数据

#### 用户调研数据

**调研方法选择：**

| 方法 | 适用场景 | 样本量建议 |
|------|----------|------------|
| 定量问卷 | 大范围验证假设 | 200+ |
| 深度访谈 | 探索性研究 | 8-15人 |
| 可用性测试 | 体验优化 | 5-8人 |
| A/B测试 | 数据驱动决策 | 足够统计显著性 |
| 焦点小组 | 需求探索 | 6-8人/组 |

#### 市场预测模型

**预测方法：**
- 趋势外推：基于历史增长率推算
- 模拟对标：对标已验证的同类市场
- 自下而上：用户数 × 转化率 × 客单价
- 多情景分析：乐观/中性/保守

**必须标注假设条件：**
- 哪些假设成立时预测成立？
- 假设不成立时的敏感性分析？

## MRD 编写的常见雷区

1. **市场机会太虚**：只说"千亿蓝海"不给数据拆解
2. **用户画像脸谱化**：没有真实场景和可验证的痛点
3. **竞品分析流于表面**：只列功能不分析策略
4. **产品建议与市场脱节**：建议的功能不是市场需要的
5. **GTM 策略不可执行**：没有预算和资源规划
6. **数据引用不靠谱**：来源不明、数据过时、口径不清
7. **缺乏决策建议**：读完不知道下一步该做什么
