# 行业分析常用框架参考

本文档汇总了 BRD/MRD 编写过程中常用的行业分析框架，供 AI 分析时参考使用。

---

## 一、市场分析框架

### 1. TAM/SAM/SOM 市场规模估算

**定义：**
- **TAM（Total Addressable Market）**：如果产品做到 100% 市场份额时的总收入
- **SAM（Serviceable Available Market）**：在我们的能力范围内（地域、渠道、价格带）可以触达的那部分 TAM
- **SOM（Serviceable Obtainable Market）**：短期内（通常 1-3 年）实际可以获取的市场份额

**估算方法：**

| 方法 | 适用场景 | 公式 |
|------|----------|------|
| 自上而下 | 有现成行业数据时 | 行业总量 × 目标细分占比 |
| 自下而上 | 数据有限时 | 目标用户数 × 客单价 × 购买频率 |
| 类比法 | 新兴市场时 | 对标市场规模 × 调整系数 |
| 价值链法 | B2B 场景 | 客户价值 × 渗透率 |

**示例（在线教育市场）：**
```
TAM = 中国在线教育市场规模 = 5,400亿元（艾瑞2024数据）
SAM = 成人职业教育细分市场 = 1,200亿元
SOM = 我们的初期目标市场（IT技能培训）= 180亿元
    短期目标市占率 = 0.5%
    短期可获取收入 = 180亿 × 0.5% = 9,000万元
```

### 2. 波特五力模型

**分析维度：**

| 力量 | 核心问题 | 分析指标 |
|------|----------|----------|
| 供应商议价能力 | 供应商能卡我们脖子吗？ | 供应商集中度、替代供应商、转换成本 |
| 买方议价能力 | 客户能压我们价格吗？ | 客户集中度、产品差异化、转换成本 |
| 新进入者威胁 | 门槛高不高？ | 资金门槛、技术门槛、网络效应、政策壁垒 |
| 替代品威胁 | 有没有更好的替代方案？ | 替代品的性价比、切换成本 |
| 行业内竞争 | 竞争激烈程度？ | 集中度、增速、差异化程度 |

**判断标准：**
- 5 力中有 3+ 力量强 → 行业吸引力低
- 5 力中有 3+ 力量弱 → 行业吸引力高

### 3. PESTEL 宏观环境分析

| 维度 | 关注问题 | 对产品的影响 |
|------|----------|-------------|
| **P** 政策（Political） | 有什么法规变化？准入门槛？补贴？ | 合规成本、准入门槛、政策红利 |
| **E** 经济（Economic） | 宏观经济走势？用户消费力？ | 用户预算、付费意愿、市场规模 |
| **S** 社会（Social） | 人口结构变化？消费习惯变迁？ | 用户需求变化、新场景出现 |
| **T** 技术（Technological） | 什么技术在成熟？什么在颠覆？ | 技术可行性、研发成本、技术壁垒 |
| **E** 环境（Environmental） | 环保要求？碳中和？ | 合规要求、运营成本 |
| **L** 法律（Legal） | 数据安全？知识产权？劳动法？ | 合规成本、风险管控 |

**使用建议：**
- 不需要 6 个维度都深入分析
- 根据行业特点选择 2-4 个最相关的维度
- 每个因素标注影响力和对业务的具体影响

## 二、竞争分析框架

### 4. SWOT 分析

**四个维度：**

| | 正面（内部能力） | 负面（内部短板） |
|---|---|---|
| **内部** | **S**trengths 核心优势 | **W**eaknesses 关键劣势 |
| **外部** | **O**pportunities 重要机会 | **T**hreats 重大威胁 |

**TOWS 战略组合（进阶）：**

| | 优势（S） | 劣势（W） |
|---|---|---|
| **机会（O）** | **SO 战略**：发挥优势抓住机会 | **WO 战略**：弥补劣势抓住机会 |
| **威胁（T）** | **ST 战略**：利用优势抵御威胁 | **WT 战略**：减少劣势规避威胁 |

**SWOT 分析质量检查：**
- ✅ 每个 SW 项都与竞品对比后有差异
- ✅ 每个 OT 项都标注了具体数据或趋势
- ✅ TOWS 组合给出了具体策略
- ❌ 只列名词不分析（"团队优秀"但没说优秀在哪）
- ❌ 内外不分（把"竞品强"放到劣势里）

---

### 5. 竞品功能矩阵

**对比维度建议：**

```
                    竞品A    竞品B    竞品C    我们
核心功能1            ✅       ✅       ❌       ✅
核心功能2            ✅       ❌       ✅       ✅
核心功能3            ❌       ✅       ✅       🔄
差异化功能           ❌       ❌       ❌       ✅
---
定价（月）           ¥99     ¥149     ¥79      ¥129
目标客户            大企业    中企业    个人      中小企业
市场占有率           35%     25%     15%      -
NPS 评分            42       35       28       -
```

## 三、用户分析框架

### 6. KANO 需求分类模型

| 需求类型 | 定义 | 用户反应 | 策略 |
|----------|------|----------|------|
| **基本型（Must-be）** | 不做就骂的需求 | 不满意 → 无所谓 | 必须做，做到行业基线 |
| **期望型（Performance）** | 做得越好越满意 | 不满意 ↔ 满意 | 持续优化 |
| **兴奋型（Delight）** | 做了就惊艳 | 无所谓 → 超满意 | 差异化机会 |
| **无差异型（Indifferent）** | 做不做都一样 | 无感觉 | 可以不做 |
| **反向型（Reverse）** | 做了反而反感 | 不满意 | 绝对不做 |

**KANO 分析步骤：**
1. 列出所有需求
2. 对每个需求设计正反两个问题（有这个功能感觉？没有这个功能感觉？）
3. 收集用户回答
4. 根据 KANO 评价表分类
5. 结合 Better-Worse 系数排序

**Better-Worse 系数：**
- Better = 满意度系数 = (A+O)/(A+O+M+I)
- Worse = 不满意度系数 = (O+M)/(A+O+M+I) × (-1)
- 坐标图四个象限决定优先级

---

### 7. 用户旅程地图（Customer Journey Map）

**旅程阶段：**
```
认知 → 考虑 → 获取 → 激活 → 留存 → 推荐
```

**每个阶段记录：**
- 用户行为（做了什么）
- 触点（在哪里发生的）
- 情绪（高兴/困惑/沮丧）
- 痛点（遇到了什么问题）
- 机会（可以优化什么）

## 四、商业分析框架

### 8. 商业模式画布（Business Model Canvas）

| 维度 | 回答的问题 |
|------|-----------|
| **价值主张** | 我们为谁解决什么问题？ |
| **客户细分** | 我们的目标客户是谁？ |
| **渠道通路** | 怎么触达客户？ |
| **客户关系** | 与客户建立什么关系？ |
| **收入来源** | 怎么赚钱？ |
| **核心资源** | 需要什么资源？ |
| **关键业务** | 需要做什么？ |
| **重要合作** | 需要谁的帮助？ |
| **成本结构** | 主要成本是什么？ |

---

### 9. RICE 优先级排序

```
RICE Score = (Reach × Impact × Confidence) / Effort
```

| 因素 | 评分标准 |
|------|----------|
| **Reach** | 影响多少用户/次/月 |
| **Impact** | 0.25最小化 / 0.5低 / 1中 / 2高 / 3极大 |
| **Confidence** | 50%低 / 80%中 / 100%高 |
| **Effort** | 需要多少人月 |

## 五、战略分析框架

### 10. 安索夫矩阵（Ansoff Matrix）

| | 现有产品 | 新产品 |
|---|---|---|
| **现有市场** | 市场渗透 | 产品开发 |
| **新市场** | 市场开发 | 多元化 |

---

### 11. BCG 波士顿矩阵

| | 高增长 | 低增长 |
|---|---|---|
| **高份额** | ⭐ 明星 | 🐄 现金牛 |
| **低份额** | ❓ 问号 | 🐕 瘦狗 |

**战略建议：**
- 明星：加大投入，抢占份额
- 现金牛：维持份额，收割利润
- 问号：选择性投入，评估潜力
- 瘦狗：退出或转型

---

### 12. VRIO 资源分析

| 维度 | 问题 | 竞争含义 |
|------|------|----------|
| **V** 价值性 | 这个资源/能力有经济价值吗？ | 是 → 可能获得竞争优势 |
| **R** 稀缺性 | 这个资源在市场上稀缺吗？ | 是 → 可能有持续优势 |
| **I** 难模仿性 | 竞争对手难以复制吗？ | 是 → 可能有长期优势 |
| **O** 组织性 | 组织能有效利用吗？ | 是 → 确实具有竞争优势 |

## 六、数据来源参考

### 可信行业数据源

| 类别 | 数据源 | 适用行业 |
|------|--------|----------|
| 综合行业 | 艾瑞咨询、QuestMobile、易观 | 互联网/移动互联网 |
| 企业服务 | Gartner、IDC、CB Insights | B2B/SaaS |
| 金融科技 | 零壹财经、亿欧、36氪 | FinTech |
| 教育 | 多鲸、芥末堆、黑板洞察 | 教育行业 |
| 医疗 | 动脉网、丁香园 | 医疗健康 |
| 电商 | 星图数据、魔镜市场情报 | 电商零售 |
| 政府 | 国家统计局、各部委 | 宏观数据 |
| 全球 | Statista、McKinsey、BCG | 全球市场 |
| 上市公司 | 巨潮资讯、Wind、同花顺 | 上市公司数据 |

### 数据验证原则

1. **交叉验证**：至少 2 个独立来源的数据相互印证
2. **时效性**：优先使用 2 年内的数据
3. **口径一致性**：注意数据口径差异（如 GMV vs 净收入）
4. **标注来源**：引用数据必须标注来源和时间
5. **置信度标注**：标注数据的可靠程度
