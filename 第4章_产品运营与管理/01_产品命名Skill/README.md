> 本节配套 Claude Code Skill：`pm-naming`

## 本节目标

用 AI 辅助产品命名决策，让新功能/新模块/新产品的命名从"拍脑袋"变成"有方法论、可评估、能说服"。

核心价值：**不是帮你定名字，而是帮你做出经得起推敲的命名决策。**

## 方法论

### Qualtrics XM 体系的启示

Qualtrics 使用"品牌前缀 + 产品类型"的命名模式：

| 层级 | 命名方式 | 示例 |
|------|---------|------|
| 平台级 | 公司品牌 + Platform | XM Platform |
| 产品线 | 品牌前缀 + 功能描述 | XM Discover / XM Engage |
| 领域应用 | 领域缩写 + 产品类型 | CX Dashboard / EX Dashboard |

核心原则：**每一层名称都传递两个信息——属于谁（品牌），干什么的（功能）。**

### 四种命名风格

| 风格 | 特点 | 适合 | 示例 |
|------|------|------|------|
| **A 功能描述型** | 一看就懂 | B 端工具、后台系统 | 订单管理中心、批量导入 |
| **B 品牌前缀型** | 建立体系 | 产品线、对外产品 | 飞鱼洞察、钉钉智能人事 |
| **C 隐喻意象型** | 有记忆点 | C 端产品、品牌功能 | 朋友圈、瀑布流 |
| **D 动作引导型** | 突出收益 | 增长功能、CTA 场景 | 一键诊断、先买后付 |

### 四维度评估框架

| 维度 | 权重 | 核心问题 |
|------|------|---------|
| 语义清晰度 | 30% | 不需要额外解释就能理解吗？ |
| 品牌一致性 | 25% | 和公司已有命名协调吗？ |
| 传播力 | 25% | 好记、好说、好搜索吗？ |
| 风险可控性 | 20% | 有歧义、撞名、文化问题吗？ |

## 使用方式

### 基础用法

```
/pm-naming
```

然后告诉 AI：
- 你想给什么起名（功能/模块/产品）
- 它是做什么的
- 已有的命名风格（如果有）

### 名称评估用法

如果已有几个候选但拿不定主意：

```
/pm-naming
帮我评估这三个名字：飞鱼洞察、飞鱼数析、运营数据助手
```

Skill 会按四维度逐个打分，输出推荐排名和风险分析。

### 示例

完整命名报告示例见 `候选命名报告-示例.md`——以"飞鱼洞察"（AI 数据分析助手）为案例，展示了从命名对象分析、风格选择、7 个候选生成、四维评分到风险审查和最终推荐的完整流程。

---

## 风险审查

每个候选都会经过四类风险检查：

| 风险类型 | 检查项 |
|----------|--------|
| 语言歧义 | 谐音、方言、缩写、中英文双关 |
| 撞名风险 | 竞品、内部项目、知名产品 |
| 商标/法律 | 商标局查询、域名、行业管制 |
| 文化/地域 | 数字忌讳、颜色/动物/符号、地域敏感度 |

高风险项一票否决，中风险标注并给出规避方案。
